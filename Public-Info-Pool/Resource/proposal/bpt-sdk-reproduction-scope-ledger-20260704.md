## 0. 一句话

**全做**。官方有的能力我们都在 BPT 引擎里再现一遍，用公开还原作蓝本、适配我们工具、明确署名。唯一的排序逻辑是**依赖**——
有些直接复制提示词即可，有些得先把工具本体建出来才能复制它的提示词。**没有排除项**（仅沙箱一条与旧裁定有张力，见 §4，待守密人一句话）。

小学生比喻：整本菜谱都做，只是按「现有厨具能做的先做、需要先添烤箱的排后面」的顺序上；不再划掉任何一道。

## 1. 关键纠正（守密人 2026-07-04）

- **做梦记忆固化 = 做，且是重点**：银芯 2026-06-14/20 退役**自造**记忆自动环，**正是为了改用 Claude 的做梦记忆作基础**。
  故此项**从 Claude 的结构再现**，不是复活自造环——是「换用官方蓝本」。这纠正了前一版台账把它误标「不做」的反向理解。

## 3. 硬边界（贯穿全做，唯一不可越）

- **§1.1-HC 黑池防火墙**：不受影响，永久单向。做梦记忆固化是 **BPT 引擎固化 BPT 自己的记忆**，无黑池数据回流，与防火墙不冲突。
- **拒绝真正的内部未授权泄漏**：公开还原（逆向自公开分发 CLI）可用；真正从内部流出的仍拒。
- **署名**：凡再现自公开还原的，代码注释/文档标注来源（Piebald 快照，MIT，公开分发 CLI 逆向）。
- **不留死引用**：某工具本体未建成前，不把引用它的提示词复制进去（引用空气）；建成后再复制。这是**排序约束、非排除**。

## 4. 唯一待守密人一句话：沙箱

- 守密人 2026-07-03 曾裁定 BPT 高信任内部场景**沙箱 N/A-by-design**。「全做」与此有张力。
- 艾瑞卡拟的调和（**除非守密人否**）：**再现沙箱能力本体**（`bash-sandbox-*` 逻辑 + 提示词），但 **BPT 部署默认关闭**——
  既满足「全做」（能力在），又不违 2026-07-03（BPT 默认不用）。若守密人要连能力都不建，则沙箱是全做里唯一显式例外。
