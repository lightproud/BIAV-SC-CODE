# BPT Agent SDK 再现范围台账：做 / 缓做 / **不做（明确标注）**

> 类型：proposal ｜ 日期：2026-07-04 ｜ 作者：艾瑞卡会话（守密人派发：「先考虑清楚哪些不做，明确标注」）
> 定位：**公开信息再现**（守密人 2026-07-04 裁定）。本台账在「先复制、尽可能实现技术部分」前，先钉死**边界**——
> 哪些官方调用点/功能我们**故意不做**，附理由，防止复制进引用不存在工具的死片段、或把 CLI 产品面误当引擎面追。
> 上游依据：官方架构推断图 `Public-Info-Pool/Resource/repo-engineering/official-cc-prompt-architecture-inference-20260704.md`（~40 调用点）；

## 0. 一句话

官方 ~40 个调用点里，只有**映射到 BPT/我们 SDK 已有工具**的那部分该复制；引用「我们没有的工具/产品面/已退役子系统」的部分**故意不做**、且**在代码与文档里显式标注为 out-of-scope**，绝不留半截引用。**做的清单会长，不做的清单更要写死**——否则复制会把官方的整个产品外壳拖进来。

小学生比喻：照着别人家菜谱做菜，先划掉所有「需要你家没有的厨具」的菜（不做、且用红笔标清为什么划），剩下的才照做——不然照抄到一半发现要用烤箱，可你家根本没烤箱，整道菜卡死。

## 1. 判定四条准则（每个「不做」必须落其一）

- **T（无此工具）**：官方提示词引用 BPT/SDK 不 ship 的工具/特性，复制进来会引用空气 → 不做（或先建工具再说）。
- **P（产品面非引擎面）**：属 CLI/桌面产品 UX，不属「agent 引擎」职责 → 不做（引擎保持通用 drop-in）。
- **S（主权无关/已裁定 N/A）**：与 BPT 主权模型无关，或守密人已裁定 N/A（如沙箱） → 不做。
- **R（银芯已退役）**：银芯自己已退役的子系统，绝不借复制之名复活 → 不做。

## 3. 硬边界（贯穿所有「做」，不可越）

- **§1.1-HC 黑池防火墙**：不受本台账任何条影响，永久单向。
- **拒绝真正的内部未授权泄漏**：Piebald 属「逆向自公开分发 CLI」的公开信息；真正从 Anthropic 内部流出的仍拒。
- **不逐字大段克隆到会引用空气的地方**：凡「不做」组的工具，其提示词片段**绝不复制进**引擎，防死引用。
- **署名**：凡再现自公开还原的，代码注释/文档标注来源（Piebald 快照，MIT，公开分发 CLI 逆向）。

## 4. 落地约定：不做的东西怎么「明确标注」

1. **代码层**：v4 及后续再现的提示词构建器，凡跳过的官方片段，在注释里写 `// out-of-scope (T/P/S/R): <reason>`，不留空引用。
2. **文档层**：POSITIONING.md §4「故意不追」清单指针本台账；本台账为 out-of-scope 单一真相源。
3. **COMPAT 层**：`docs/COMPAT.md` 对官方特性的行，凡本台账判「不做」的，标 `OUT-OF-SCOPE(理由码)` 而非「待做」，防被误当缺口去追。

---

**总台账**：做的是「映射到已有工具的核心引擎纪律 + 安全胶水 + 审查/验证流水线 + 记忆/提示词组装」；不做的是「无此工具的 Workflow、整个产品 UX 面、已裁 N/A 的沙箱、银芯已退役的做梦记忆」。**边界先写死，复制才安全。**
