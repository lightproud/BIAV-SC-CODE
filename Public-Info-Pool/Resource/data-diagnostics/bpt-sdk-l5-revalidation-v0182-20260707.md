## 3. 官方臂失分归因：KD-L5-01（/tmp 散落物，官方 CLI 自身怪癖）

官方臂的 13 分失分几乎全部源于**已登记的已知差异 KD-L5-01**——官方 Write 强制绝对路径，Haiku 猜 `/tmp/<文件名>`，散落物污染后续 repeat，判 `FAIL(success)`（终态 success 但产物落错位置/被前轮污染）。逐条实据（取自作业日志）：

- **code-03**：官方 r1–r5 **全部**留 `/tmp/fizz.mjs` → 5/5 全 FAIL → **官方 0/5**（我方 5/5）。
- **code-05 r2**：官方留 `/tmp/result.txt` → FAIL → 官方 4/5。
- **longconv-01 r1/r4/r5**：官方留 `/tmp/note.txt` → 3 次 FAIL → 官方 2/5。
- **longconv-02 r3**：官方留 `/tmp/sum.txt`；r2/r5 亦 FAIL(success) → 官方 2/5（KD-L5-01 + KD-L5-02 注入过敏方差）。

**这不是我方引擎的功劳缺口，是官方 CLI 的路径锚定怪癖**——我方对应题全 5/5。

> 小学生比喻：官方那位同学习惯把作业纸随手塞进公共抽屉，下一次考试翻出上次的旧纸就写错了；银芯每次都把纸放回自己桌肚，所以不受影响。

## 5. 效率轴（只记分、不进门禁）：我方普遍更省更快更少轮

Gate B 只看通过率；效率是附带观测。逐题看，我方引擎在**成本 / 轮数 / 墙钟**三轴上几乎全面占优：

- **成本**：典型如 retrieval-01 $0.0059 vs $0.0249（省 ~76%）、code-01 $0.0110 vs $0.0258、code-04 $0.0183 vs $0.0324。两臂合计 **$2.1920**。
- **轮数**：如 code-04 5 vs 12、retrieval-03 3 vs 9、code-02 2 vs 5——我方更少轮达成同结果。
- **墙钟**：如 code-01 6100ms vs 20928ms（~3.4× 快）、code-04 10990 vs 21727。
- **缓存诊断**：两臂均 **scenario a**（缓存真生效，设计点）——bpt writes=97,026 / reads=5,369,299；官方 writes=150,937 / reads=6,869,466。我方缓存写更少（前缀更精简）、命中同样健康。

> 小学生比喻：同一批题，银芯用更少的草稿纸、更少的步骤、更短的时间做完，还做得一样对——省钱又快。
