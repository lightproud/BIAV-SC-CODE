# BPT Agent SDK — 中文 i18n 成本调查(为何 v0.28.0 贵了 ~50%)

- **日期**:2026-07-08
- **触发**:v0.28.0 中文化 i18n 战役(#523–#533,把工具描述 + 系统提示 + 分类器提示译成中文)后,Haiku L5 全量轮花 **$3.28 vs v0.18.2 的 $2.19(+50%)**;bpt 缓存读从 5.4M 涨到 16.8M(3.13×)。
- **方法**:两个 `count_tokens` 探针($0,不生成)+ L5 作业日志 per-run 缓存明细对照。工具:`tests/integration/token-probe.mjs`(跨模型)+ `token-probe-perfile.mjs`(逐文件),CI `token_probe` dispatch。
- **一句话结论**:**成本主因 NOT「中文提示更大」。** 译文内容只膨胀 **1.41×**,远解释不了 2.2–3.2× 的成本/缓存读膨胀;差值在**每轮被重读的对话上下文**(主嫌:中文思考/输出的累积),不在静态提示。

---

## 0. 两次被数据翻掉的假设

| 假设 | 结果 | 依据 |
|------|------|------|
| **H1「Haiku 用旧分词器、中文吃亏,换新模型就省」**(守密人初判)| **证伪** | 跨模型探针:同段中文,Haiku 1.75 tok/字,Sonnet-5/Opus-4.8/Fable-5(共用新分词器)1.89 tok/字——**Haiku 反而更省**,换新模型只会更贵 |
| **H2「中文提示更大 → 前缀翻 3 倍」**(艾瑞卡一度下的结论)| **证伪** | 逐文件 count_tokens:8 个译文文件全在 1.2–1.45×,缓存前缀内文件合计 **1.41×**,没有任何一个是元凶 |

守密人两次「不对劲」都对;艾瑞卡据实纠正。

## 1. 跨模型分词器对照(H1 证伪)

同段真实中文工具描述,按各模型自身分词器 `count_tokens`(run 28934024667):

| 模型 | EN tok | ZH tok | ZH 每 CJK 字 | ZH/EN |
|---|---|---|---|---|
| **claude-haiku-4-5** | 11,209 | **18,588** | **1.75** | 1.66× |
| claude-sonnet-5 | 15,551 | 20,046 | 1.89 | 1.29× |
| claude-opus-4-8 | 15,551 | 20,046 | 1.89 | 1.29× |
| claude-fable-5 | 15,551 | 20,046 | 1.89 | 1.29× |

- Sonnet-5 / Opus-4.8 / Fable-5 数字**逐字节相同 = 共用一套「新」分词器**;Haiku 用另一套。
- 对**中文**,Haiku 反而**更省**(18,588 < 20,046)。对英文 Haiku 更是省得多(11,209 vs 15,551)。
- **「换新模型省中文成本」不成立**,方向相反。

## 3. 那 3.2× 在哪:对话级重读(缩圈 + 主嫌)

L5 per-run 缓存明细,同任务同轮数对照(bpt 臂):

| 任务 | 轮数 | v0.18.2 读 | v0.28.0 读 | 倍数 | 写(两轮) |
|---|---|---|---|---|---|
| document-01 | 3 | 62,498 | 199,444 | 3.19× | ~1360 / ~1352 |
| document-02 | 3 | 62,025 | 198,962 | 3.21× | ~同 |
| longconv-01 | 5 | 103,560 | 331,792 | 3.20× | ~1090 / ~1090 |
| longconv-02 | 4 | 82,673 | 265,279 | 3.21× | ~830 / ~830 |

- **轮数相同、写几乎相同,唯独读 ~3.2×。** 静态前缀只 1.41×,扣掉前缀后 **ZH 的对话历史读量约是 EN 的 ~6×**。
- 差值只能在**每轮被重读的对话上下文**(助手输出 + 工具结果,逐轮累积)。

**主嫌:中文思考/输出的累积。** claude_code preset 默认开思考(E1);中文提示 → 模型**用中文思考与作答** → 中文 token(~1.75/字)在对话里越滚越厚 → 每轮重读反复计入 → 缓存读滚成 3.2×,尽管静态前缀只有 1.41×。

> 小学生比喻:课本(静态提示)只厚了四成;但学生被中文课本带着**用中文打草稿**,草稿纸越写越厚、每翻一页还要把前面所有草稿重读一遍——让「读书电费」翻三倍的是越滚越厚的**中文草稿**,不是课本。

**诚实残余(§4.2)**:「中文思考累积」是**主嫌、非终证**。终证 = 拆两轮的助手输出 / thinking token 体量(L5 报告 artifact 的 per-run usage);本报告未做该拆分。

## 4. 影响与可选优化

- **正确性无涉**:gate B 未回归(bpt 87/90 ≈ 88/90,详见 `bpt-sdk-l5-revalidation-v0182-20260707.md` 与 v0.28.0 轮)。本调查只关成本。
- **成本量级**:+50%(L5 轮 $2.19→$3.28);缓存读单价低($0.1/MTok haiku),单会话绝对额小、规模化才积累。
- **优化方向(供裁定,均需 L5 复验 + 触及 i18n 裁定 B)**:
  1. **内部系统提示留英文**:模型读英文指令照懂,**面向用户的输出仍可中文**——既砍「中文思考/输出累积」的大头、又不损用户可见的中文体验。内部提示对用户不可见,是「最大成本 + 最大行为风险 + 零用户收益」的一块。
  2. 若要保留中文内部提示,则成本主要来自**思考累积**,压缩静态提示(缩写译文)**收效有限**(静态前缀只占 1.41× 那一小截)。
