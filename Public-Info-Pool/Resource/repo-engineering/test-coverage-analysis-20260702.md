# 测试覆盖率分析与改进提案（2026-07-02）

> 类型：repo-engineering / 生成会话：test-coverage-analysis
> 测量方式：本地全量运行 `pytest tests/ --cov=scripts --cov=projects/news/scripts --cov=projects/wiki/scripts --cov-branch`
> 测量口径与 `.coveragerc` 三源根一致；CI 门槛现行 `--cov-fail-under=85`（`.github/workflows/test.yml`）。
> 本档为分析提案层（弱约束），既有测试策略权威见 `docs/testing-strategy.md`。

## §2 改进提案（按优先级）

### P1 — 补测 `compact_discord_archive.py` 的写盘主路径（58.5% → 目标 90%+）

未覆盖的恰是**最危险的部分**：`main()` 的全量文件遍历 + `process_file` 非 dry-run 分支
（临时文件写入 + `os.replace` 原子替换）。该脚本的职责是就地重写全量 discord 档案
（2026-06-22 精简行动，工作树 3.4G→2.0G）——若未来再次动用而写路径有回归，损坏的是
全量档案层本体。建议：monkeypatch `DISCORD_DIR` 指向 tmp 目录造 3-5 个 JSONL 样本，
断言 (1) dry-run 零写盘字节不变；(2) 实写后记录数不变、恒留字段无损、可被
`discord_compact.expand_record()` 还原；(3) 中途异常不留半截临时文件。

> 小学生比喻：这位搬家师傅要把全家家当拆包重装，但他只在纸上演练过「怎么算省多少箱子」，
> 真正的搬运动作一次都没排练——第一次实战就是动真家当。

### P2 — 消除 `deliverable_path.py` 的覆盖率盲区（0% 测量值 → 真实可见）

`tests/test_deliverable_path.py` 有 21 条测试且全部通过，但全部经 `subprocess.run` 驱动
CLI，coverage 不跨进程，测量恒为 0%。该脚本是 §6.2 产物路径的**强约束守卫**（挡同义
分裂 / 形式漂移），值得让它的真实覆盖进入门槛保护。两个方案任选：

- **方案 A（推荐）**：测试改为 import 模块直接调函数（`compute_path` / `register` 等），
  保留 2-3 条 subprocess 冒烟测试验 CLI 入口。改动小、变异测试将来也能覆盖。
- 方案 B：启用 coverage 子进程测量（`COVERAGE_PROCESS_START` + `.pth` 钩子），侵入
  CI 环境配置，维护成本高，不推荐。

> 小学生比喻：这个学生每次考试都参加、都及格，但试卷交在隔壁考场，成绩单上永远登记
> 「缺考」——要么让他回本考场考，要么去隔壁把分数抄回来；回本考场更省事。

### P3 — 处置 `measure_discord_compaction.py`（0%、无测试）：先问退役，否则补测

只读测量器，其历史使命（为 2026-06-22 精简裁定提供真实数字）已完成。同时它内嵌了
一份与 `projects/news/scripts/discord_compact.py` **平行的精简规则副本**——规则若在
discord_compact 侧演进，此档静默漂移。二选一，需守密人裁定：

- **优先：退役删除**（结论已落盘于精简裁定，工具用完即弃）；
- 若保留：补单测钉住其精简规则与 `discord_compact.COMPACT_*` 常量**逐字段一致**
  （防两份规则漂移），外加 `--limit` 抽样与统计聚合的小样本断言。

> 小学生比喻：装修完留在客厅的卷尺——要么收进工具箱（删），要么每年校一次刻度别让它
> 和新买的尺子对不上（钉一致性）；最糟的是既不收也不校、摆着落灰还可能量错。

### P4 — 拆分 `report_render.py`：纯逻辑与重依赖解耦（60.3% → 纯逻辑部分 95%+）

`render()` 是一整块：frontmatter 切片、`## §0` 起始定位、`◇ ◇ ◇` 分隔符替换、h2 锚点
TOC 正则重写这些**纯字符串逻辑**，与 markdown 转换、weasyprint PDF 写盘这些**重依赖
动作**焊死在同一函数。weasyprint 按「渲染依赖按需装」立场不进 CI（既定裁定，不动），
导致纯逻辑部分陪绑失测。建议提取 `build_html(body_md, title, ...) -> str` 纯函数
（markdown 库很轻可进 CI），`render()` 只留「调 build_html + 写 PDF」薄壳。TOC 锚点
编号、divider 替换、h1 切除三段逻辑当前全裸奔——它们出错时产出的是排版损坏的正式报告。

> 小学生比喻：打印机（weasyprint）家里没有可以理解，但写作文和排版是自己的手艺，
> 不该因为「没打印机」就连作文都不检查了——把作文和打印拆开，作文照常批改。

### P5 — 补 `discord_archiver.py` 错误分支（42 个缺失分支）

缺的集中在：论坛帖 starter 抓取异常吞掉（`logger.debug` 后继续）、消息分页 API 失败
break、限速/时间片用尽提前退出等**降级路径**。这是每小时无人值守跑的采集器，错误分支
恰是它在真实网络环境最常走的路。建议用可编程假 `_api`（按调用序列抛异常/返回异常形状）
钉住：单帖失败不拖垮整轮、失败后 last_id 不越过 starter（源码注释里的雪花号陷阱）、
时间片耗尽的干净收尾。**不必追 100%**——按 testing-strategy 反凑数立场，只钉「失败被
正确隔离」这一不变量。

> 小学生比喻：送报员晴天的路线练得很熟，但下雨天绕路、狗追、门铃坏这些情况全靠临场
> 发挥——而送报员的日常恰恰一半是雨天。

### P6 — CI 覆盖率门增加分支维度（`--cov-branch`）

现行门只看行：语句 94.8% vs 分支 89.4%，5.4 个百分点的差距全是「行跑到了、分支没拐
过去」的死角（P5 那类错误分支正是行门看不见的）。建议 `test.yml` 加 `--cov-branch`，
门槛数字维持 85 不动（合并口径 93.5%，余量充足）——只是让仪表盘显示真实的路口通过率，
不收紧闸门，与「不逼凑数测试」的既定立场不冲突。

> 小学生比喻：现在只检查「每条街都走过没有」，不检查「每个路口两个方向都拐过没有」——
> 加分支维度就是把路口也画进巡逻图，巡逻范围不变、地图更诚实。

### P7 — 给 `.githooks/pre-push` 上验证程序（当前零测试）

该 git 钩子是 lesson #28/#34/#39（413 胖包连环坑）的**唯一自动化防线**，纯 shell、
零测试。它做的是「push 前把分支基底对齐最新 origin/main」这类有真实行为语义的操作，
一旦悄悄坏掉，防线失效的发现方式将是再踩一次 413。建议一条 pytest：tmp 目录里
`git init` 造本地 + 裸远端双仓，安装钩子后分别验证（1）基底陈旧时被对齐、（2）基底
新鲜时不动、（3）钩子非零退出能拦下 push。约 40 行测试换一条防线的在册保证。

> 小学生比喻：家门口的防盗门装好后从没试过还锁不锁得上——每年演习一次开关门，
> 别等真有小偷才发现锁芯早锈死了。

### P8 — 变异测试作用域扩容一名候选：`discord_compact.py`

`expand_record()` 是紧凑 schema（§5.2「缺字段 = 默认值」）的**唯一还原器**，全量 732 万
条社区档案的可读性都压在它的默认值表上。它满足 setup.cfg 既定收录标准：逻辑密集、
常量表噪声低、存活体可明确归因。建议按既有模式补 `tests/test_mut_discord_compact.py`
包路径孪生档并加入 `[mutmut]` 作用域。行覆盖已有（test_discord_compact_unit），此举
验证的是断言真的钉住了每个默认值——正是变异测试设计来抓的盲点类型。

> 小学生比喻：压缩包的解压器只有一个，全部家当都用它开箱——不光要确认它跑过一遍，
> 还要故意拧错它一颗螺丝、看检查员能不能发现。

## §3 刻意不提的事（与既有策略对齐）

- **不建议上调 85 门槛**：testing-strategy §1 的反凑数论证成立，本档从其立场。
- **不建议给采集器网络路径堆 mock 追 100%**：P5 只钉失败隔离不变量，不铺凑数面。
- **wiki 前端 TS 层（数据桥/config.mts）暂不建 JS 测试栈**：数据桥现按 #221 裁定导出
  空数组，`npm run docs:build` 冒烟已够；W2 重建可信基线后再议（届时值得一条
  「数据桥输出 schema」断言）。
- **shell 部署脚本（silver-mem-*.sh 等）不追覆盖**：低频人工操作，投入产出比不合算；
  P7 单独点名 pre-push 是因为它是无人值守防线，性质不同。

## §4 建议落地顺序

P2（半天、消灭最大数字失真）→ P1（半天、消灭最危险空白）→ P3（守密人裁定后半小时）
→ P6（一行改动）→ P4 / P5 / P7 / P8（各半天到一天，可分会话逐个领走）。
全部落地后预计：语句覆盖 96.8%+、分支覆盖 92%+，且门槛保护范围回到与实际脚本面一致。
