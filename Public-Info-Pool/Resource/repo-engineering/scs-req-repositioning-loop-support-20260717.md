## 1. 定位声明(终态)

银芯 BIAV-SC 是黑池五层介质架构中的**引擎层**:最慢的介质、版本化、可替换的商品泵。它的职责由三条否定与一条肯定完整定义:

1. **不认识斜杠。** 引擎不拦截、不解析、不展开任何以 `/` 开头的文本约定。用户输入长什么样、如何触发能力,是客户端层的 UX 决策。
2. **不持有钟。** 引擎不含任何跨 query 的定时、轮询、编排逻辑。钟属于 runner 守护层(宿主进程)。
3. **不追宿主面。** 引擎的宿主侧 API(query 签名、options、编排语义)按黑池需求自主演化,不以官方 Claude Code / claude-agent-sdk 的宿主侧形状为义务。
4. **只做一件事:把一次结构化的 agent 调用执行到位,并把模型侧表面保持在模型训练分布附近。**

模型侧表面(工具名、参数 schema、系统提示词习语)持续贴近官方形状——受益者是模型本身的调用稳定性,而非替换退路。直连 Messages API 的轻快响应是核心价值:任何提案若代价是引入 CLI 式中间层开销,直接拒绝。

## 3. 需求范围:循环支撑接口面

背景:黑池将于 runner 守护层自建无人值守 loop(周期轮询监控/采集,通宵常驻级)。引擎的义务是让该 loop **可被外部零补丁地干净构建**,而非自己成为 loop。交付以下六项:

### R1 续轮注入 API
公开 API:向一个已存在的会话追加一条新的用户轮并继续执行,返回该轮完整结果。要求:
- 与现有 sessions/resume 机制统一,不另起会话抽象;
- 宿主可在注入前读取会话的上下文占用与累计成本;
- 每轮注入支持附带结构化前置内容(供 R4 ledger 注入)。

### R2 预算事件流
在既有 maxBudgetUsd 基础上补结构化事件面:
- `budget:threshold`(可配置比例,如 80%)与 `budget:exhausted` 两级事件;
- `budget:exhausted` 触发时,当前轮完成后停止,并产出一份**收尾报告**(累计成本、轮次数、最后状态摘要),以结构化对象交给宿主;
- 事件通过引擎既有 hook/事件机制暴露,宿主订阅即可,不需轮询。

### R3 压缩钩子(保留区)
自动压缩发生时,允许宿主声明**不可丢弃的结构化上下文区**:
- 宿主注册的保留区内容原样进入压缩后的上下文;
- 保留区有大小上限并在超限时向宿主报错,而非静默截断;
- 典型用途:R4 的已报异常清单必须活过压缩,去重语义不能托付给有损摘要。

### R4 ledger 原语
去重账本的纯逻辑,引擎级测试纪律罩护:
- 记录已报告事件(键 + 时间戳 + 摘要)、查询是否已报、按容量/时限淘汰;
- 可序列化/反序列化(宿主自选存储介质,引擎不做持久化);
- 提供与 R1"结构化前置内容"和 R3"保留区"的适配函数,一行代码接入。

### R5 LoopControl 工具定义
模型侧表面,引擎出品并测试:
- 工具形状(初稿,以 chase 结果修正):`LoopControl { action: "propose_stop", reason: string }`;
- 语义:模型只能**提议**终止,提议以结构化事件抛给宿主,循环是否继续由宿主决定;引擎不因提议而改变自身行为;
- 该工具作为可选导出,由宿主在装配 loop 会话时注册;
- chase 前置(唯一一项):对照官方当前版本的命令工具面 schema(工具名、参数、命令列表注入方式),使 R5 形状贴近训练分布。若语料暂不可得,按上述自研 schema 先行,并在 COMPAT 参照笔记中标注"待对齐"。

### R6 引擎版本声明面
运行时可查询的引擎表面版本声明(引擎版本、工具面清单及各自版本),供热更新能力层(技能 md、MCP 清单)做加载时兼容校验。配套:在技能编写标准中增补 frontmatter 引擎兼容声明字段的规范条目。

## 4. 斜杠退役(引擎行为变更)

1. 引擎不再对 prompt 文本做任何斜杠命令识别与展开;query 收到的 prompt 一律原样进入装配。
2. 既有 /loop、/goal 相关解析器与控制器(parseLoopCommand、createPromptLoop、goal 状态机等)**不删除**,降级为可选导出工具包:测试全部保留,文档标注"供宿主客户端做输入解析时选用,不构成引擎行为"。
