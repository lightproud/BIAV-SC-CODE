## 一、结论先行

**健康度：优。** 这是银芯迄今工程完成度最高的产物——1414 单测全绿、类型与构建零错、
技术债近乎为零、定位与 §1.1-HC 防火墙同向合规。它已从「概念验证」跨到「可发货、被真实
消费方（BPT Desktop）依赖」的成熟阶段。主要风险不在代码质量，而在**它天生要追一个移动的
上游**（官方 SDK）带来的永久维护税。

> 小学生比喻：这个项目像一台自己造的、跟原厂遥控器长得一模一样的万能遥控器——按键布局
> 照抄原厂（drop-in），但里面的电路是自研的、不塞原厂那颗被禁的芯片。造得很扎实，唯一的
> 累是原厂每出新款、你就得跟着对一次按键。

## 三、架构与定位（合规审计）

- **自研引擎，直驱 Messages API**：`fetch` + SSE，**不打包 CLI 子进程**——这正是它的存在
  理由（让 BPT Desktop 脱离被禁的 `claude.exe`）。主循环 / 工具派发 / 权限门 / hooks /
  MCP / 会话持久化全部本包实现。
- **定位合规**：`docs/POSITIONING.md`——银芯自有工程产物、**银芯→黑池单向输出物**，
  方向与 §1.1-HC 防火墙一致（黑池数据从不回流）。审计通过，无越界口子。
- **提示词来源**：默认系统提示词为官方 Claude Code 提示词的**「公开信息再现」**（源自 MIT
  许可的逆向重构公开件、明确署名），非自撰、非内部泄漏件。守密人 2026-07-05 已把定位从
  clean-room 反转为「公开信息再现」并落 `memory/decisions.md`——此为已裁定项，非遗留风险。

> 小学生比喻：不塞原厂被禁的芯片 = 不调 `claude.exe` 那个被禁的子进程，自己直接跟总部
> （API）通话；且严守单向邮筒规矩——只往黑池寄，黑池的东西绝不回流进来。

## 四、保真度（vs 官方 SDK）

- **表面等价 ~90%+**，一致性测试金字塔 L1–L5 全部封顶（流语法 / 选项语义 / 工具差分 /
  故障注入 / 端到端统计带）。首轮真 API 端到端 L5 **两臂打平 48/54 (88.9%)**，且本 SDK
  几乎每任务更便宜更快（缓存对齐后成本差拉到 ~3.6× 我方省、速度 2.8×）。
- **COMPAT 台账**（`docs/COMPAT.md`）：**FULL 55 / PARTIAL 39 / ACCEPTED 10 /
  UNSUPPORTED 7**——drop-in 承诺有 7 项硬缺口 + 39 项部分支持，不是 100% 无缝。
- **漂移哨兵在岗**：官方 `agentSdk 0.3.201 / claudeCode 2.1.201` 双钉隔离
  （`tests/conformance/pins.json`），周 cron 只报不追，已成功抓到并追平 0.3.199→0.3.201。

> 小学生比喻：行为保真像两个学生做同一张卷子，分数打平（88.9:88.9），但我方交卷更快更省纸；
> 差的那几题主要不是引擎笨，是「要不要开思考」这类模型档位没对齐，已在 E1 批修掉。

## 五、风险与债务（诚实清单）

1. **移动靶维护税（结构性，无法根治）**：项目价值绑定「追官方」，官方每发新版就需对齐
   一轮。哨兵能报，但追不追、追多深仍需守密人逐次裁定——这是永久成本，非一次性。
   > 小学生比喻：抄的是别人还在改的作业，人家改一笔你就得跟一笔。
2. **总线因子 = 1**：全部由艾瑞卡会话推进，无第二维护者。四件文档（CONTEXT / COMPAT /
   ARCHITECTURE / CHANGELOG）齐全缓解了知识断层，但仍是单点。
3. **自查自合无 CI 硬门**（守密人 2026-06-21 裁定）：验证责任移交会话本身。本次评估即复现
   了此风险的另一面——状态权威档曾停在「668 测试」而真实已 1414，**状态数字滞后于真实**。
   纪律靠人守，非靠闸守。
4. **`total_cost_usd` 是静态价表估算**：官方调价后会偏，非实时计费。
5. **drop-in 有 7 项 UNSUPPORTED**：迁移方需先查 COMPAT 台账，不能盲切。
