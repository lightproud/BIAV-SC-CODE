## 1. L5 首轮解剖（run 28736460533：repeat=3，108/108，乙门禁打平 48/54 vs 48/54）

四个任务级失败点（我方 chat-03 0/3、code-01 0/3；官方 code-03 1/3、longconv-02 0/3）
解剖为三个系统变量与一个度量伪影：

- **S1 思考不对称**：官方 CLI 默认开 extended thinking（54/54 留痕含 `thinking_tokens`
  事件，合计 1161 次），我方引擎默认关。我方两个失败点的共同真因——chat-03 我方 8 token
  膝跳抢答把「逆字母序」答成正序；code-01 我方只排序漏偶数长度（官方带思考也仅 2/3，
  判据本质是「自发想到偶数长度」的概率）。
  比喻：官方答题前默认打草稿，我们的默认口算抢答。
- **S2 /tmp 锚定 + 跨 repeat 污染**：官方 Write 强制绝对路径，Haiku 把新档案猜到
  `/tmp/<名>` 而非任务目录；repeat-1 的遗留物经官方读前写门把 repeat-2/3 引进死胡同
  （现成正确文件消灭了 ENOENT 自救信号）。官方 code-03 r2/r3 与 longconv-02 r3 的真因。
  比喻：作业本落在鞋柜，下一个考生看到现成作业直接在鞋柜上改——老师只查书桌。
- **S3 官方安全姿态过敏**：官方系统提示词注入警惕 × Haiku，把良性中文「记住：甲的值是
  12。只需确认」判为提示注入直接拒绝（longconv-02 r1/r2，中英两种拒绝文案；上下文
  实际跨轮保留完好）。
- **M1 度量伪影**：官方臂流式多轮每轮各发一个 result，runner 只取末个 → 报表 turns=1。
  「中文多轮提前终局」系误读——官方没有提前散场，是记分员看错行。

副产物：官方 Write 读前写门语义实锤（仅拦已存在未读文件、新建不拦、报错文案原文），
直接成为 E4 的对齐规格。

## 2. 测试用例层加固（PR #453）

  直接看到既有解释。

## 4. 产线 SSE 故障闭环（PR #461）

- **故障**：BPT 产线报「流中断 原因：Malformed SSE payload for event "(none)"」。
- **诊断链**：银芯指认 "(none)" 指纹 =「无 event 行 + 非 JSON data」不可能出自原生
  Anthropic 端点 → 给 BPT 三步核实清单 → BPT `curl -N` 抓原始字节**铁证**：idealab 网关
  `/api/anthropic` 端点带 OpenAI 方言遗留——流尾追加 `data: [DONE]`、错误帧无 event 行。
  官方客户端 message_stop 即收工不碰尾卡；我方读到流关闭才撞上。
- **修复**（`src/transport/anthropic.ts`）：① message_stop 即收工（官方同款生命周期，
  尾卡根本不进解析器）；② stop 前无 event 名非 JSON 帧跳过（debug 留片段）；③ 有 event
  名坏帧照抛且带现场（前 120 字符 + 已解析帧数 = E6b）。5 条回归测试。
  比喻：给新邮差上两课——正课「看到签收章就走人别翻信封底」，备用课「没名没姓又读不懂
  的卡片扔了记一笔，别喊包裹损坏」。
- **后证**：完备性推进中把同款方言送进 L4 双臂差分，`[DONE]` 尾卡两场景双臂全绿
  ——修复在差分下成立。

## 5. 完备性四缺口清零（PR #463，全程零真实 API 开销）

1. **环境保真轴（从无到有）**：仿真器新增 `sse-gateway` 脚本类（逐字节复刻 idealab 抓包
   形状），L4 +3 场景双臂对跑。抓到**真发现 KD-L4-05**（两轮稳定）：官方 2.1.201 不认
   无名错误帧，报「empty or malformed response (HTTP 200)」并**重试一次**，再把失败编码成
   assistant 文本 + `result/success`；我方 `isErrorPayload` → `APIStatusError(400)` → 干净
   `error_during_execution` 零重试。**产线实义：官方 claude.exe 在 idealab 后面遇 API 错误
   会重试并谎报成功；我方快速失败带真错误类型。**
   比喻：考砸的卷子，官方学生擦掉分数写「已完成」交回去；我们的学生如实报错在哪。
2. **MCP 差分首批（L3 扩容）**：arm.mjs `buildOptions` 加每臂 SDK 句柄，各臂用自家
   `tool()`/`createSdkMcpServer` 建进程内服务器。ping / 软失败 isError **双臂
   CONTENT_MATCH（tool_result 编码逐字一致）**；抛异常 / 未注册工具为措辞级差
   （KD-L3-22/23，两轮稳定）。zod schema 语义留第二批。
3. **Fix-2**：`run-l5 --thinking=N` 双臂同预算钉死 + workflow `l5_thinking` dispatch 输入
   ——「引擎差 vs 思考差」可一键拆分。
4. **聚合自证**：度量规则抽成 `l5-aggregate.mjs`，vitest 喂首轮真实官方 result 序列锁定
   ——测试工具自己有了测试。

## 8. 战略意义（断供换装线）

自研引擎对官方引擎现在持有**活体差分证据**的三项优势：

1. **更准**：同任务集同模型 92.6% vs 79.6%（乙门禁只要求不差 5pp，实际反超 13pp）；
2. **更省**：全轮总开销省 2.25×（$0.355 vs $0.801）；此前 vs-official 对齐重跑另有
   速度 2.8× 与缓存命中打平的存证；
3. **错误处理更诚实**：产线网关方言下我方快速失败带真错误类型，官方重试并 success 化
   API 错误（KD-L4-05）——对黑池这种跑在企业网关后面的环境，这是可用性级差异。
