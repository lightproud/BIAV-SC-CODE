## 第一部分：档案评价（一手采访 + 设计决策 + 银芯仓库轨迹）

### 1. 救火型制作人——履历事实

Light 并非项目创始人。项目 2019 年由初代制作人创立；2021 年末濒临解散
（开发瓶颈 + 成本压力 + 核心玩法不清 + 市场预期被原神抬高），Light 以第二制作人身份加入，
做了三刀：砍高成本探索玩法、砍过度复杂的碎片化叙事、重心从调查转向战斗，
游戏在此次重启中正式定名 Morimens。此后经历 2023 年下半年「关服前抢先上 Steam」
（获「特别好评」）与 2024 年末核心成员离职的第二次危机，团队至今仍在重建。

**推断**：核心能力是在资源枯竭局面下做减法并保住项目命脉。两次濒死、两次撑过来，
非单次运气可解释。

### 2. 设计者——原则清晰且肯为原则付账

三条核心原则在案：玩法与叙事统一 / 每个选择有意义 / 只增强不削弱（削弱必退款）。
两个可验证「原则非口号」的实例：

- **角色 24**：四生态位压进一个角色，采访原话承认「商业上不划算」，
  只为服务「人格分裂」叙事——战斗设计师「为理想燃烧一次」。
- **疯狂量表机制**：概念克苏鲁味十足，但因认知负担过重、与守密人
  「主动直面不可名状之物」的角色身份冲突而砍掉，简化为症状卡。
  砍自己喜欢的东西比做加法更能说明设计纪律。

### 3. 运营者——错误记录在案且自己承认

采访中主动列出的失误：命轮重复需求初期定太高、后期加码系统（超越晓明 / 额外等级上限 /
Final Axiom）造成心理负担、第一年削弱角色伤害体验、第一年社区沟通太随意导致批评被放大。
最后一条演变为自述「最艰难的一课」：疲惫 + 处理批评不当 → 一度考虑辞职 →
结论「最好的救赎是把游戏做好」。补救有实据：v2.0 SSR 概率 1%→3%、硬保底 80→30，
承诺关服时提供离线版（预算已预留、架构保持可分离，参考 SINoALICE 先例）。

### 4. 守密人——银芯仓库直接观测

- **裁定果断且留痕**：决策日志每条有日期、理由、覆盖关系。2026-06-15 发现 wiki
  占位数据长期误导引用，裁定整层清空重建——宁归零不留假数据。
- **肯砍自己造的东西**：自建记忆系统 9 模块约 3,410 行代码，发现与平台原生定位冲突后
  2026-06 连代码带数据整套删除。
- **防火墙纪律一以贯之**：黑池 → 银芯永久关闭的硬约束（§1.1-HC）由其本人裁定并层层加固。

## 第二部分：社区侧交叉验证（全量档案层）

### 方法

- 关键词命中（制作人 / 制作组 / 开发组 / dev team / developer / producer）：
  **4,444 条**——Discord 4,216、Steam 78、Google Play 43、微博 42、App Store 38、
  Reddit 14、其余平台个位数。
- 社区昵称「**校猫**」（英文社区称 The Cat）：Discord 2,190 行、Steam 评测 61 条、
  微博 90、Bilibili 22——命中量远超正式称谓，社区把他当具体的人而非匿名官方谈论。
- 计数由 ripgrep 直接产出（R2 纪律），随机抽样人工阅读约 80 条原文。
  关键词法有漏网（不点名的讨论抓不到），结论按方向性信号采信。

### 发现一：英文社区——罕见的正面浓度

- Reddit（2026-06-01）：「我们有对社区最透明的游戏制作人……Light 的决定不总让所有人满意，
  但他确实落实了我们要求的东西」
- Discord（2026-04-18）：「我喜欢 Light，他是我最爱的 failboy CEO 兼制作人。
  他和团队顶着压力、背着债把这游戏带给我们」
- App Store（2026-04）：「玩过的最好的抽卡游戏，开发组极其讲理、毫无贪婪」；
  Steam 俄语评测（2026-03）：「开发者听玩家意见，几乎是实时改」

负面同样在案：2025-08「星辰篇感觉就是开发者放弃了这游戏」、2024-08「制作人说过的话
没兑现被玩家点名」、2026-05 Reddit 记录「Ethereal Cores 风波」沟通争议。
正面为主、批评具体，整体为健康信任关系。

### 发现二：中文社区——爱恨交织，骂声集中且有明确时间簇

- **爱恨同体固定句式**（均出自好评）：「游戏好，校猫坏」（2026-01）、
  「校猫不发病绝对是个好游戏」（2024-11）、「不要死啊校猫」（2024-10）——
  骂着他打好评是中文社区主流姿态。
- **极端差评有明确时间簇**：2024-12 前后（数值 / 削弱风波期）出现「校猫一天不死」级
  差评；2026-04 前后（自抽号限制、界域数值限制）再起一簇。
- **正面深度评价并存**：TapTap（2026-06）「深刻体会到校猫对玩家和对这款游戏的良苦用心……
  每个小版本结束就发问卷……官方补偿说实话很大方，但对氪金玩家的伤害确实弥补不回来」。
- 区域温差被社区自己点破——Discord 中文原声（2025-12）：「英区真的很爱校猫，
  他现在这么膨胀有很大原因是英区的人在惯着他」。

### 发现三：与档案自述的对账——自我评价诚实度获证实

采访中 Light 主动承认的四项失误（第一年削弱 / 后期系统加码 / 命轮需求过高 /
社区沟通随意），与社区骂声最集中的主题**逐项对得上**：2024-12 中文差评簇骂的正是
数值与削弱，Reddit 记录的风波正是沟通问题。他没有回避社区最痛的点。

微博侧佐证：2026-05 起官方反复置顶【制作人公开信】「你们曾拯救了忘却前夜——
如今我们给出未来结局的承诺」，与采访中的离线版承诺、「最好的救赎是把游戏做好」一脉相承。

## 综合判定

Light 是一名**危机中接手、以减法立身、原则可验证、失误肯留痕**的制作人，且
**高曝光、高透明、争议随决策实时波动**：

1. 英文社区给予近乎罕见的信任溢价（透明度 + 慷慨度是两大高频关键词）。
2. 中文社区以「校猫」之名进行高强度日常审判，但审判底色是「怕游戏死」而非「盼游戏死」。
3. 采访中的自我批评经全量社区数据对账**未发现粉饰**。
4. 风险面清晰：数值与沟通是最易再次失分的两个位置——社区档案显示这两处的旧伤疤，
   玩家都还记得。

他最特殊的角色雷梦娜——「复制上一张打出的卡牌 = 从过去经历中获得力量」，
及其亲自加进 V1.1 PV 的台词「无论失败多少次，我终将拯救你」——档案显示，
这大概也是他对这个项目本身的自述。
