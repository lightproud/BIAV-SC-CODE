## 6.2 关键缺口

### 缺口 A：角色→技能→数值的映射链断裂

AwakerConfig（角色基础）和 AwakerPotency（技能潜力）之间缺少关联索引。当前状态：
- 知道有 72 个角色
- 知道有 1,035 条技能数据
- **不知道哪些技能属于哪个角色**

需要：从 GameConfig_References 或其他配置中找到 AwakerId -> PotencyId 的映射关系。

### 缺口 B：敌人/Boss 数据完全缺失

Lua 提取中没有独立的敌人配置表。Stage.lua 可能内含敌人引用，但未被解析。这使得战斗平衡分析（如DPS check、护盾需求）无法进行。

### 缺口 C：造物（遗物）系统

探索中获取的造物（Artifacts/Relics）是 Roguelite 体验的核心组成，但没有独立数据表。可能分散在 Item.lua 或 GameConfig_References 中。

### 缺口 D：Lua table 的参数化数值

AwakerPotency 中的 `[Arg1]`、`[Arg2]` 等占位符表明技能数值是参数注入的。完整的数值公式需要交叉引用 numeric_config.txt 或 GameConfig_References.lua 才能还原。

### 缺口 E：版本差异数据

当前 Lua 提取是 v2.4.0 的单一快照。无法回溯早期版本的数值变化（如 v1.x 的旧概率、被调整的角色平衡）。制作人提到的"第一年削弱后回退"等事件无法从数据层面验证。

## 6.3 未实装角色信号

AwakerConfig.lua 中发现的未实装/预备角色是重要情报：

**"本源"变体体系**：本源汀克特（已实装）、本源·奥吉尔、本源萝坦、本源沉睡之主——暗示"本源形态"可能成为未来的角色变体方向（类似已有的"环行"、"熔毁"、"血链"、"诞妄"前缀）。

**神话实体角色**：诺登斯（伟大深渊之主）、撒托古亚（旧日支配者）、兰提戈斯（燃烧之神）、夏塔克鸟、黑法老（奈亚拉托提普化身）、亚弗戈蒙——如果实装，代表游戏将直接引入克苏鲁神话中的高阶实体作为可玩角色。

**制作人已确认的未公开角色**：采访中提到"一个超维界域创世纪角色，概念已定稿但未公开"——可能对应 AwakerConfig 中某个占位条目。

## 6.4 建议

### 短期（数据清洗）

1. **构建 Lua 解析管道**：编写 Python 脚本将 24 个 Lua table 解析为 JSON 格式，建立统一的 ID 索引
2. **建立角色-技能映射**：从 GameConfig_References 中提取 AwakerId -> PotencyId 的关联，补全角色技能链
3. **分离已实装/未实装角色**：根据 Summon.lua 中是否存在对应卡池配置，区分 AwakerConfig 中的已实装角色和预备角色

### 中期（数据分析）

4. **密契属性池完整映射**：从 Item.lua 中提取所有密契套装及其属性池组合
5. **卡池时间线可视化**：利用 Summon.lua 的 ID 顺序还原完整的卡池发布时间线
6. **语音文本交叉分析**：Voice.lua 的 2,562 条语音文本可用于验证和补充叙事分析

### 长期（持续追踪）

7. **版本差异追踪**：每次游戏更新后重新提取 Lua 表，diff 对比发现数值变化和新增内容
8. **未实装角色监控**：追踪 AwakerConfig 中占位条目的数据填充情况，作为新角色预告的早期信号
