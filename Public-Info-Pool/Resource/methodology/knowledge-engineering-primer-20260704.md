## 一、问题形态：你的病，银芯得过

「什么都接上了、但没有结构」不是黑池特有。银芯 2026 上半年踩过同型的坑：

- **自造记忆环之死**：银芯曾自建会话蒸馏 + TF-IDF 语义召回 + 知识图谱 + MemRL + 做梦自动环，
  2026-06 整套退役删除——与平台原生记忆定位冲突、维护成本吞掉产出（`memory/decisions.md` 相关条）。
- **占位数据之祸**：wiki 结构化层曾以合成占位充数（24/72 全 fixture），长期误导引用，
  2026-06-15 守密人裁定**整层清空重建**。
- **路径漂移之乱**：产物路径「每次会话各编一套」，同日报出四种写法，2026-06-21 以脚本强约束根治。

小学生比喻：银芯也曾是那间书堆在地上的图书馆，还自己发明过一台不好用的自动找书机器人。
下面三味药是拆掉机器人、装上书架之后总结的。

## 二、总纲：神经符号白盒骨架（先读这个再动手）

权威源：`memory/knowledge-layer-design.md`（知识层北极星，全文必读，本节只给梗概）。

**一句话**：结构化知识层 = 有结构的概念网络，承载**白盒**知识（可枚举、可审计、可测试、
可程序操作），与 LLM 承载的**黑盒**知识（会泛化、会脑补）互补而非竞争。三器官分工：

| 器官 | 承担 | 干别人干不了的 |
|------|------|--------------|
| 骨架（概念网络） | 结构 / 身份 / 层 / 边界 | 可枚举可审计可测试 |
| 结缔组织（全文搜索） | 钻「只指不放」的血肉 | 模糊、无界、按内容命中 |
| 肌肉（LLM） | 推理 / 泛化 / 生成 | 处理没见过的（有脑补代价） |

三条工程命令（治理混乱的总纲，逐条有银芯实测背书）：

1. **白盒只花在骨架上**——结构进概念层，血肉（海量正文/二进制）留给搜索与指针。
   判据：一个概念若没有非平凡的边、也没有搜索给不出的元数据，它就是「被冻进文件的搜索结果」。
2. **把不变量测起来**——白盒是唯一能写测试的知识：指针不落空、数据层标注、生成幂等，全部上 CI。
3. **死守覆盖哨兵**——白盒的招牌死法是**假完备**（地图看着全、其实只和策展一样全）；
   边界必须真，未覆盖要么建层要么显式豁免。

比喻：骨架是书架标签系统，搜索是沿标签进书堆翻页，LLM 是那位博闻强记但偶尔记串的馆员——
三者缺谁都不行，但**把整本书抄到标签上**（白盒放本体）和**让馆员凭记忆管库存**（黑盒管结构）都是病。

## 三、第一味药：OKF 概念层（给知识一副骨架）

权威源：`CLAUDE.md §6.1` + 生成器 `scripts/build_okf_bundle.py` + 成品 `okf/`（12 层 296 概念，可直接翻看）。

- **格式**：一概念一 markdown 文件，YAML frontmatter 存元数据（唯一必填 `type`），
  `graph.json` 存带类型的关系边。Open Knowledge Format v0.1 开放规范，零专有依赖。
- **三条铁律**：(1) 一概念一文件；(2) **放指针不放本体**——除唯一本体层外全部只持 `resource`
  指针，2.1G 归档绝不复刻进概念文件（<4KB/个）；(3) **数据层标注不可互换**——每个指针带
  `data_layer` 标签（full_archive / output / curated），防「把抽样当全量」（见 §五 lesson #30）。
- **两层结构（Pillar A 教益，2026-07-04）**：skeleton（真有边、实测连通）与 search-tier
  （有意孤立、靠搜索可达）**显式分层并测试连通率**。教益原话：**诚实分类 > 强行连接**——
  真数据 derive 不出的边不要硬造（同画师边 = 零区分度噪声星，已剪）；孤立不是缺陷，未声明的孤立才是。
- **运行时消费**：静态索引 `okf/kb_index.json`（`scripts/build_kb_index.py`）+ MCP 四原语
  `kb_search` / `kb_get` / `kb_neighbors` / `kb_overview`（后端 `scripts/kb_navigator.py`，
  import-only 库）——LLM 顺关系图逐跳导航、按需取概念，不整仓灌上下文。
- **生成物治理**：bundle 是可复现生成物，源变更未重建会被 `tests/test_kb_governance.py` 拦下；
  纪律测试 `tests/test_okf_bundle.py` + `tests/test_okf_pointer_layers.py` 守铁律。

比喻：每本书一张卡片（不是把书抄一遍），卡片之间拉真实存在的线（不是为了好看乱拉线），
馆员查书先摸卡片再顺线走，最后才进书堆。

## 四、第二味药：4R 归位（给文件一个家）

权威源：`CLAUDE.md §6.2` + `memory/strategy/repo-slimming-plan.md §8` + 守卫脚本 `scripts/deliverable_path.py`。

- **4R 本体**：Root / **Resource**（正式产物，按主题类型分目录）/ **Record**（原始时序档案）/
  **Reference**（引用资料）+ 第 5 个 R-sibling **Rough**（即兴草稿，默认不进版本库，人工晋升）。
  注：4R 出自 BPT v2.0，2026-06-21 被定性为银芯↔黑池接口协议——黑池侧本就有此本体，直接对齐。
- **按性质分流**：可检索 text 进 git（保 grep / diff / AI 可分析），二进制进 Releases（零访问损失）。
  银芯实测反例：text 进 Releases 后被迫写还原脚本下 301MB 才能建索引。
- **弱约定升强约束**：路径规则不靠文档自觉，靠脚本确定性算出
  （`Resource/{类型}/{主题}-{YYYYMMDD}[-rN].{ext}`）；主题类型走**开放注册表**（`types.json`：
  形式定死、清单可增、near-match 挡同义分裂）。根因：任何 prompt 级约定都是弱约束
  （2026-05-19「入口反转」裁定），会话临场编路径必然漂移——**规则要么进代码，要么等着被违反**。

比喻：家里定「玩具收进右边柜子」没用，孩子每天放的位置都不一样；有用的是柜子上装一个
只对准玩具形状开口的投掷孔——放错了根本塞不进去。

## 五、第三味药：确定性索引家族（让海量档案可查而不常驻）

权威源：三个同家族实现，全部**构建期生成、零 ML、零常驻服务**：

| 索引 | 覆盖 | 实现 |
|------|------|------|
| `projects/news/index/community_index.json` | 732 万条社区档案按平台×月聚合（量/语言/词典法情感/高频词/覆盖） | `scripts/build_community_index.py` |
| `projects/wiki/data/processed/story/story_search_index.json` | 1026 条剧情 lore 倒排表 + 单元画像 | `scripts/build_story_index.py` |
| `okf/kb_index.json` | 概念层倒排表 + 邻接表（运行时导航底座） | `scripts/build_kb_index.py` |

- **设计共性**：倒排表（词→文档）+ 预聚合画像；分词用**领域词典 FMM**（`scripts/silver_tokenizer.py`，
  词典自举自项目专名，不引 ML 分词器）；索引只答「去哪看」，全文钻取回落 ripgrep 原文件。
- **为什么零 ML**：确定性 = 可测试 + 可复现 + 免维护常驻服务。银芯的语义检索子系统（TF-IDF/向量）
  已整套退役——对策展规模的知识层，词典法 + grep 的检索质量够用，而运维成本差一个数量级。
- **异常可见性**：时序索引带基线偏离信号（如 `vol_index` = 本月量÷前 6 月中位数），采集断崖一眼可见。

比喻：不养一位随叫随到的图书检索员（要发工资、会请假、答案还时对时错），而是每次进新书后
重印一本目录册——印目录的机器谁按都出同一本。

## 六、踩坑移植清单（银芯替你交过的学费）

权威源：`memory/lessons-learned.md`（编号对应）+ `memory/decisions.md`。逐条一句话 + 比喻：

1. **数据层混用（lesson #30）**：把输出展示层 16 条当全量 5,455 条分析，得出 0.27% 抽样率假命题。
   对策 = data_layer 标注硬约束。比喻：把货架样品数当库存总数。
2. **占位数据之祸**：合成 stub 一旦入库就会被当真引用，清理成本远超当初「先占个位」省下的。
   对策 = 宁缺勿假，基线只认一手数据。比喻：账本上不能写「大概有钱」。
3. **档案与现实脱节（lesson #29 模式）**：决策记了档案没改、或档案改了决策没记，反复发生。
   对策 = 对账测试（银芯有 CLAUDE.md 三卫 + 日期哨兵抓跨档漂移）。比喻：课程表改了不通知班级，全班照旧走错教室。
4. **弱约束本质（2026-05-19）**：prompt 级文档约束必然衰减；硬约束只存在于工具层与平台层。
   对策 = 关键纪律全部脚本化 / 测试化（§四投掷孔同款）。
5. **信息无损 ≠ 使用无损（2026-06-22）**：压缩档案时删「恒默认字段」省 41% 零便利损失；
   再删 timestamp 虽可从 id 还原、但毁掉「grep 即查」的便利——**到此为止**。
   比喻：扔掉包装盒可以，把说明书也扔了就得每次上网查。
6. **生成物必须有治理测试**：源与生成物的一致性靠 CI 拦，不靠记性（本仓 `test_kb_governance.py`
   在本档入库当天就真实拦下过一次漂移）。比喻：门口装个「出门前检查带没带钥匙」的蜂鸣器。
7. **先骨架后花活**：银芯自造记忆环（图谱/MemRL/做梦）死于「检索花活先于结构地基」；
   扩散激活这类高级检索被排在两层结构（Pillar A）之后才动工。比喻：地基没打就装旋转楼梯。

## 七、落地路线建议（从混乱开始的顺序）

以下顺序是银芯路线（北极星 §七 A→B→D→C→E）对「从零治乱」场景的改写，黑池侧按自身形状裁剪：

1. **裁定权威源**：每类知识指定唯一权威档案/系统，其余降级为视图（银芯例：进度数字只活在
   project-status，其余档案只指针不复刻）。
2. **标数据层**：全量 / 输出 / 策展三层先标清，混用即报错。
3. **4R 归位**：文件先有家，路径规则脚本化。
4. **最小概念层**：只给真有结构价值的东西建概念（判据见 §二命令 1），孤立参考层诚实标 search-tier。
5. **确定性索引**：倒排 + 聚合画像，构建期生成，检索回落 grep。
6. **运行时导航**：索引之上加 LLM 可调用的导航原语（search / get / neighbors / overview）。
7. **治理测试 + 覆盖哨兵**：幂等、指针不落空、源-生成物一致、覆盖有豁免台账——全部进 CI。
8. **然后才是花活**：扩散激活等高级检索，等骨架连通了再说。

**反顺序警告**：银芯的死亡路线是 8 → 4（先建自动记忆环再补结构）；活下来的路线是 1 → 8。
