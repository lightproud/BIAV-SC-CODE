V2.5.1「沙耶之歌」联动窗口（2026-05-18 → 05-26）

本清单从全量档案层提取本窗口社区报告的技术故障与缺陷，按模块整理为标准缺陷单 （模块概述 / 重现步骤 / 实际结果 / 预期结果）。每条标注来源与状态。

状态口径（遵 §6.11 R3 区分「官方确认」与「玩家报告」）： - 官方已修复 — V2.5.1.1 补丁公告或 official-q-a 标「已解决/Resolved」 - 官方处理中 — official-q-a 标「处理中/Processing」 - 官方已回应 — official-q-a 标「已回答/Answered」，多为非代码问题或给出规避方案 - 玩家报告 — 仅见玩家反馈，窗口内未见官方确认（信度 C，需研发侧复核）

纯数值 / 付费 / 设计争议（如卡池合并、无垢升级不计抽数、累计百万目标）不属故障范畴， 已剔除，另见社区分析报告。

A 模块 · 客户端稳定性与性能

本窗口最高频的技术故障类别。崩溃 / 卡死 / 黑屏 / 加载失败集中在移动端，多条已随 V2.5.1.1 维护修复，部分加载与发热问题仍有零星反馈。

A-1 关卡内卡死 + 疯狂闪退 官方已修复

  • 重现步骤：iPhone 13 / iOS 26.4.2，进入关卡战斗，连续作战一段时间后触发。
  • 实际结果：画面卡死——可点退出、点物品、点角色，但角色无法在格子间移动；点「修复」按钮无效，需强退重进。
  • 预期结果：关卡战斗全程可正常操作格子移动，不发生卡死。
  • 来源 / 状态：official-q-a 玩家报告（含设备信息）→ 标「已解决」（05-19）。

A-2 高画质高帧率发热闪退 官方已回应

  • 重现步骤：iOS 设备开启高画质 + 高帧率，进入超维(Ultra Realm)、Boss 本，或释放高特效大招（如环行拉蒙娜、克莱门汀），或通关后观看剧情。
  • 实际结果：设备过热并闪退。
  • 预期结果：长时间高负载场景不因发热崩溃。
  • 来源 / 状态：official-q-a → 官方回应「设备过热所致，建议在设置中调低画质/帧率改善」（05-19，硬件层规避，非代码修复）。

A-3 启动后 Logo 消失即黑屏 官方已修复

  • 重现步骤：更新游戏后（多为长期未登录的回流玩家），iPhone 打开 App，开发商 Logo 显示后停在黑屏。
  • 实际结果：长期停在黑屏，重启 App、清除缓存均无效；玩家因怕丢号不敢卸载重装。
  • 预期结果：Logo 后正常进入登录/加载流程。
  • 来源 / 状态：Reddit（05-26）+ official-q-a（05-20）多源 → 关联 official-q-a「Stuck on Loading Screen」标「Resolved」（05-25）。

A-4 开头加载卡「网络繁忙」 官方已修复

  • 重现步骤：Google Play 下载安装后首次进入，登录 Google 账号开始下载资料，挂后台再回来。
  • 实际结果：一直卡在 54%，提示「网络繁忙」；关闭重开网络、改连 WiFi、清缓存、清数据、重启手机（ROG 7 Ultimate）均无效。
  • 预期结果：资料下载正常完成进入游戏。
  • 来源 / 状态：official-q-a（05-25）→ 标「已解决」；另有玩家自发规避「开全局代理」可解（Google Play ★5）。

A-5 登录进入后崩溃 / 新玩家持续崩溃 官方已修复

  • 重现步骤：登录后进入游戏主界面阶段（新玩家高发）。
  • 实际结果：进入时崩溃 / 反复崩溃。
  • 预期结果：登录后稳定进入。
  • 来源 / 状态：official-q-a「post-login crash on entering」「Constant Crashes (New Player)」→ 标「Resolved」（更新后修复）。

A-6 Faded Legacy 第8章与终章 Watchpoint 事件触发卡死 官方已修复

  • 重现步骤：进入 Faded Legacy 第8章或终章，触发 Watchpoint 事件。
  • 实际结果：游戏卡死。
  • 预期结果：Watchpoint 事件正常结算。
  • 来源 / 状态：V2.5.1.1 补丁公告 [Combat]→Stage，明示「Fixed」。

A-7 觉醒连点导致卡死 玩家报告

  • 重现步骤：战斗中对角色觉醒（如莱克）连续快速点击。
  • 实际结果：直接卡死，只能重开。
  • 预期结果：连点不应导致客户端卡死。
  • 来源 / 状态：中文综合频道玩家提及（05-23）→ 未见官方单独确认，建议研发复核连点防抖。

A-8 阿拉克涅超维在手机端持续卡顿 玩家报告

  • 重现步骤：手机端进入阿拉克涅(Arachne) Ultra Realm。
  • 实际结果：持续卡死/卡顿；同账号在 iPad 上不卡。
  • 预期结果：移动端各机型表现一致，不卡顿。
  • 来源 / 状态：泰语频道（05-23）→ 玩家报告，疑机型/性能相关。

A-9 加载失败 / 完全无法进入（多端） 玩家报告

  • 重现步骤：Steam PC 端 / 移动端启动游戏。
  • 实际结果：Steam「游戏根本加载不出来」；俄语区「画面很美但完全无法运行，根本进不去，怎么都没用」；App Store「总是中途无响应」。
  • 预期结果：正常加载进入。
  • 来源 / 状态：Steam(en/05-21)、Steam(ru/05-23)、App Store(en/05-18)→ 玩家报告，部分与 A 模块其他项重叠。

A-10 界面操作卡顿明显 玩家报告

  • 重现步骤：常规浏览游戏界面、执行操作。
  • 实际结果：界面复杂、操作繁琐、卡顿明显。
  • 预期结果：界面操作流畅。
  • 来源 / 状态：Steam 负评(schinese/05-21)→ 玩家报告，性能/交互体验类。

B 模块 · 登录 · 账号 · 区服可达性

涉及账号安全与区服可达性，潜在影响面大，部分（账号丢失）属高危需优先核查。

B-1 俄区必须开 VPN 才能登录 玩家报告

  • 重现步骤：俄罗斯地区直接启动游戏（不开 VPN）。
  • 实际结果：无法连接/进不去；需开 VPN 加载完成后再关掉才能玩。玩家明确「以前不开 VPN 也能用」。
  • 预期结果：俄区可达性与此前一致，无需依赖 VPN。
  • 来源 / 状态：Google Play(ru/★5/05-18)+ 俄语频道多条（05-19/05-22）→ 玩家报告，疑近期区服/合规(RKN)变化，建议核查。

B-2 港区 Google 账号无法登录 玩家报告

  • 重现步骤：港区设备，可正常打开 Google Play 更新，但进入游戏登录 Google 账号。
  • 实际结果：一直「无法登录谷歌账号」。
  • 预期结果：港区 Google 账号正常登录。
  • 来源 / 状态：Google Play(zh/★4/05-24)→ 玩家报告。

B-3 多个账号绑定邮箱后数据丢失 玩家报告 ⚠高危

  • 重现步骤：账号绑定邮箱（疑「Google 商店手游绑定邮箱后再填 Google 邮箱」流程）。
  • 实际结果：多个账号绑了邮箱后「直接没了」。
  • 预期结果：绑定邮箱不应导致账号数据丢失。
  • 来源 / 状态：Google Play(zh/★1/05-18)+ 游戏问题频道相关疑虑（05-18）→ 玩家报告，单条但属数据安全高危，建议优先复现绑定链路。

B-4 Steam 端登录卡在登录画面 玩家报告

  • 重现步骤：Steam 下载后尝试登录。
  • 实际结果：一直卡在登录画面，长时间不跳转（附截图）。
  • 预期结果：正常完成登录。
  • 来源 / 状态：Reddit(en/05-24，含截图)→ 玩家报告。

B-5 账号找回（已自助解决，记录在案） 官方已回应

  • 重现步骤：玩家无法登录原账号，提供付费记录请求找回。
  • 实际结果：最终玩家逐一登录其它游戏账号后自行找回，非系统故障。
  • 预期结果：—（误报）。
  • 来源 / 状态：official-q-a → 标「Resolved」。官方口径：账号/付费问题应登录任一账号后联系客服。记录用途：佐证账号找回流程对玩家不够清晰。

C 模块 · 战斗与玩法机制

关卡、Traphase、超维(Astral Reign)、列车模式等玩法逻辑缺陷。多条随 V2.5.1.1 修复， 少数机制异常仍在处理。

C-1 灰烬遗迹「感应」后刷新黑印消耗异常 官方已修复

  • 重现步骤：超维「灰烬遗迹」中，未携带「致群星」命轮时使用「感应」，再刷新商店；随后第二次使用「感应」。
  • 实际结果：首次后刷新价格错误归零（不消耗黑印）；第二次「感应」后刷新反而增加黑印。
  • 预期结果：刷新按规则正常消耗黑印。
  • 来源 / 状态：official-q-a（05-18）+ V2.5.1.1 补丁公告明示「Fixed」→ 标「已解决」（05-19）。

C-2 Astral Reign 5-15 非首领战不支持自动战斗 官方已修复

  • 重现步骤：进入 Astral Reign 5-15 的非 Boss 战斗。
  • 实际结果：不支持自动战斗。
  • 预期结果：仅 Boss 战不支持自动战斗，其余支持（修复后行为）。
  • 来源 / 状态：V2.5.1.1 补丁公告 [Combat]→Stage，「Fixed」。

C-3 Traphase 开局缺失起手选牌阶段 官方已修复

  • 重现步骤：进入 Traphase 对局，特定情形下对局开始。
  • 实际结果：起手选牌阶段缺失，玩家开局无手牌；另「显示/隐藏牌」按钮在起手阶段错误显示紫色块。
  • 预期结果：开局正常进入起手选牌，按钮显示正常。
  • 来源 / 状态：V2.5.1.1 补丁公告 [Combat]→Traphase，「Fixed」。

C-4 Traphase「感知扭曲」状态缺持续时间说明 官方已修复

D 模块 · 灵工坊 · 命轮 · 抽卡记录

D-1 灵工坊天赋描述错串成其他觉醒体 官方已修复

  • 重现步骤：英文客户端查看被近期加强角色的灵工坊(Soulforge)天赋资质描述，切换不同 Soulforge 等级。

  • 预期结果：各等级显示该觉醒体自身天赋描述。
  • 来源 / 状态：official-q-a（05-18，多人复现/含截图）+ Reddit「Soulforge bug」（05-18）→ 标「Resolved」（05-21）；另「Daffodil's SoulforgeAptitude bug」同期「Resolved」。

D-2 抽卡记录未显示完整6个月 官方处理中

  • 来源 / 状态：official-q-a「Echoing wishes incorrect rules display」→ 标「Processing」。

D-3「诞生仪式」词缀在技能页无描述 玩家报告

E 模块 · UI · 显示 · 交互

红点、动画、占位文本、文本溢出/重叠、统计空白等界面层缺陷。多为视觉/交互瑕疵， 个别（邀请码卡死）影响可操作性已修复。

E-1 邀请码界面连点导致菜单死锁 官方已修复

  • 重现步骤：在邀请好友活动/邀请码界面快速反复开关点击。
  • 实际结果：菜单硬卡死，所有按钮失效（可稳定复现）；中文侧记为「邀请活动卡死」。
  • 预期结果：界面快速操作不导致死锁。
  • 来源 / 状态：official-q-a 中英双语多报（05-18，含 UID）→「Invite interface bug」「[Reproducable] Menu hard lock hung」标「Resolved」（05-19/05-21）。

E-2 幽灵红点无法消除 官方已回应

  • 重现步骤：UI 更新后日常登录。
