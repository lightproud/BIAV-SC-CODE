# 环行记（Ring Chronicle）— 设计文档

> 忘却前夜（Morimens）衍生同人玩法 · 吸血鬼幸存者（Vampire Survivors-like）
> 受限层项目 · 仅引用公开可查阅信息 · B.I.A.V. Studio 银芯知识层
> 最后更新：2026-06-03

## 1. 体验定义（Experience Definition）

### 1.1 一句话

> 你是**守密人**，在融蚀（Erosion）吞噬世界的最后时刻，操控一名**唤醒体**于环行之地抵抗成群涌来的融蚀造物，靠不断**觉醒命运卡**变强，看自己能在这片正在被遗忘的世界里坚持多久。

### 1.2 命名

「环行记」取自唤醒体「**环行·拉蒙娜**」——其正典机制为「复制上回合最后打出的卡牌 = 从过去经历中获得力量」（来源：制作人 2026-04 采访「玩法与叙事统一」理念）。环行 = 轮回、回响、不断重来——与吸血鬼幸存者「一局局重开、每局滚雪球」的体验内核同构。

### 1.3 核心情绪曲线

| 阶段 | 时长 | 玩家体验 |
|------|------|---------|
| 开局脆弱 | 0–2 min | 武器单一、被融蚀怪逼近，紧张 |
| 滚雪球 | 2–6 min | 命运卡叠加、清屏感成型，爽 |
| 共鸣成型 | 6–9 min | 纯界域共鸣触发，构筑质变 |
| 终焉考验 | 9–10 min | 融蚀领主降临，险胜或陨落 |

### 1.4 设计准则（继承制作人三大理念）

1. **玩法与叙事统一**：每把武器对应一名唤醒体的正典能力（反击 / 触腕 / 复制 / 胚胎），不是贴皮。
2. **让每个选择都有趣**：每次升级给 3 张命运卡，跨界域 vs 纯界域共鸣形成有意义取舍。
3. **只增强不削弱**：命运卡只做加法；平衡靠敌人成长，不靠削玩家。

## 2. 设计方案（Design）

### 2.1 核心循环

```
移动躲避  →  武器自动攻击  →  击杀掉落灵知（Gnosis）
   ↑                                      ↓
存活计时  ←  选 1/3 命运卡（升级） ←  灵知充满 → 升级
```

- **操作**：WASD / 方向键移动；武器全自动；唯一主动决策是升级时三选一。
- **胜利**：存活满 10 分钟并击败融蚀领主。
- **失败**：生命归零。

### 2.2 界域 = 武器家族（四系，源自真实四界域机制）

| 界域 | 配色 | 真实核心机制 | 武器化设计 |
|------|------|-------------|-----------|
| **混沌** Chaos | `#8b5cf6` | 死亡抵抗 / 银钥共鸣 / 可与任何界域协同 | **回响刃**：环绕角色旋转的银钥，复制上一击伤害 |
| **深海** Deep Sea | `#3b82f6` | 触腕自动攻击 / 纯深海加倍触腕 | **触腕**：自动锁定最近敌人发射的追踪弹 |
| **血肉** Flesh | `#ef4444` | 胚胎融合 / 吞噬 / 反击 | **荆棘场**：贴身持续伤害光环，击杀回血（吞噬）|
| **超维** Hyperdimension | `#eab308` | 卡牌复制 / 额外回合 | **超维爆发**：周期性全屏放射状能量波 |

### 2.3 纯界域共鸣（真实 team_rule v1 机制移植）

- 装备武器若**同属一个界域且数量 ≥ 3**，触发该界域**纯色共鸣**全局增益。
- 混沌特性：银钥共鸣——混沌可与任何界域协同，共鸣门槛 −1（移植「可与任何界域协同」正典）。
- 这制造「专精一系拿共鸣」vs「混搭吃灵活」的构筑取舍（准则 2）。

### 2.4 可玩唤醒体（首发 4 名，每界域 1 名，取自真实名录）

| 唤醒体 | 界域 | 正典依据 | 初始武器 | 被动天赋 |
|--------|------|---------|---------|---------|
| 环行·拉蒙娜 | 混沌 | 复制上回合卡牌 | 回响刃 | 武器伤害每 30s 自增 2%（回响积累）|
| 图鲁 | 深海 | 侍奉外域存在（星芒篇关键）| 触腕 | 投射物 +1 |
| 潘狄娅 | 血肉 | 反击叠加、多次伤害（fixture 数据最全）| 荆棘场 | 受击反弹 15% 伤害 |
| 朵尔·熔毁 | 超维 | 第三部核心、独立个人故事 | 超维爆发 | 爆发范围 +20% |

### 2.5 敌人（融蚀造物）

| 名称 | 行为 | 出现 |
|------|------|------|
| 蚀仆 Husk | 直线扑向玩家，量大 | 全程 |
| 速影 Wisp | 高速但低血 | 1 min 起 |
| 壅肿 Bloater | 高血慢速、近身爆开 | 3 min 起 |
| 织雾者 Weaver | 远程吐蚀息弹 | 5 min 起 |
| **融蚀领主 Erosion Lord** | Boss，召唤小怪 + 冲撞 | 10 min 终战 |

### 2.6 命运卡（升级池）

- **新武器**：解锁未持有的界域武器（最多 6 把）。
- **武器进阶**：等级 +1（伤害 / 数量 / 范围 / 频率随武器类型成长）。
- **被动遗物（Trinket）**：源自真实 trinket 概念——生命上限、移速、全局伤害、范围、冷却缩减、拾取范围、回血。

### 2.7 数值骨架

- 玩家：HP 100、移速 130 px/s、拾取范围 60 px。
- 升级曲线：`xpToNext = 5 + level*4`（指数缓增）。
- 敌人成长：血量与生成密度随 `time` 线性放大，承接「只增强不削弱：靠敌人成长平衡」。

## 3. 技术架构

```
projects/game/
├── DESIGN.md            # 本文档
├── index.html           # 游戏外壳 + UI + canvas
├── build.mjs            # JSON 配置 → config.js（供 file:// 双击运行）
├── config/              # 配置层（JSON，唯一事实源）
│   ├── characters.json  · weapons.json · enemies.json
│   ├── waves.json       · upgrades.json · realms.json(软链概念)
│   └── config.js        # build.mjs 生成，浏览器全局加载
├── src/
│   ├── core.js          # 纯逻辑（node + 浏览器共用，可测）
│   ├── sprites.js       # 程序化 Canvas 精灵
│   ├── render.js        # 渲染层（浏览器）
│   ├── input.js         # 键盘输入（浏览器）
│   └── main.js          # 装配 bootstrap
└── test/
    └── core.test.mjs    # node 测试：伤害/升级/共鸣/生成
```

- **核心逻辑环境无关**：`core.js` 不依赖 DOM / fetch，node 与浏览器共用，保证可 headless 测试（准则：目标驱动执行）。
- **配置即 JSON**：`config/*.json` 为唯一事实源；`build.mjs` 生成 `config.js` 让游戏 `file://` 双击即玩，无需起服务器。
- **美术程序化**：`sprites.js` 用 Canvas 绘制，按界域配色，自包含、零外部图。
