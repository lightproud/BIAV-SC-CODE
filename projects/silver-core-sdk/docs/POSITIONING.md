## 0. 一句话定位

**Silver Core SDK = 一套稳定兼容的调用「表面」 + 一个我们完全掌控、比原版更简单更可靠的独立「引擎」。**
不是「官方的克隆」,是「恰好共享一套表面的独立产品」。

**深层理由(2026-07-04 拷问修订,升格)**:独立自有引擎(独立重实现、不复制专有代码)不是 claude.exe 被禁的应急替代,
而是 **BPT 元工具层主权的结构性要求**——跨国技术商业竞争与地缘政治交错、数据主权宣示、
集团内部技术路线摇摆、乃至公司本身可能被交易导致基建整体更换,这些条件使「依赖外部工具」
本身成为定位级风险。禁令只是提前引爆点,不是原因。定位全景见 `memory/bpt-positioning.md`。

## 1. 两条轴:表面 vs 行为(务必分开谈)

谈「与官方的差距」,先分清是哪条轴——混谈必得错误结论。

| 轴 | 指什么 | 能否靠追更闭合 |
|----|--------|--------------|
| **表面完整度** | 有没有那个 Options 字段 / Query 方法 / 消息变体 / 工具 | **能追**,持续交税 |
| **行为保真度** | 同一 prompt,模型是否做出同样的决策、同样的手感 | **结构性封顶,永不追平** |

决定「换装后用户会不会觉得不对劲」的是第二条;大多数人误以为是第一条。

**参照系精确化(2026-07-04 拷问修订)**:官方 Claude Code 从未在办公环境大规模可用,
团队的参照体验不是「官方最新版」,而是**禁令时点的旧内核 BPT 体验**(旧 BPT 内核即官方
claude agent sdk / claude.exe 子进程,官方行为经它真实到达过用户)。行为保真度风险因此
**保持原级**(换装可感知),但对照目标从**移动靶变成固定靶**——旧内核已禁、版本冻结,
不再周更漂移。这给「钉基线」第二重依据:不必追官方新版行为,只需贴住旧内核手感;
「贴住固定靶」是 §7 A/B 测量线可实际度量的目标——七任务代表集的对照基线应明确定义为
旧内核行为。

## 2. 永久性技术细节损失(秘方层 + 直连的结构性代价,做不到而非没做)

1. **系统提示词**——官方 preset `claude_code` 映射的是庞大、闭源、频调的真实
   系统提示词;本 SDK 默认映射的是官方主循环提示词的**忠实开放再现**(v5,取材自公开重构
   Piebald-AI 快照,MIT/逆向自公开分发 CLI,已提为 `claude_code` preset 默认)。残余缺口
   不再是「守净室纪律永远补不平」,而是**秘方层**:公开重构 ≠ 官方线上活体提示词 + 未文档化
   CLI 行为,叠加**模型/provider 主权**(换模型换手感)。故这仍是**残余**缺口(已收窄、未彻底闭合),
   但性质是「秘方层 + 主权选择封顶」,不是自设的禁抄规则。
2. **未文档化行为**——文档描述表面,CLI 有海量未公开行为(压缩确切触发、缓存键
   卫生、重试看门狗、33 变体精确语义、子代理调度)。文档没写的对不上。
3. **工具语义实现差**——官方 Grep=ripgrep、Read 渲图像/PDF、Bash 有 AST 规则+沙箱;
   我们是纯 JS 正则 / 纯文本 / 前缀规则。**理论可闭合,非结构性,可调**。
4. **`auto` 是模型分类器**,我们是静态启发式——学不像模型的判断。

## 3. 为何拒绝「始终追平官方」作为北极星

1. **赢不了的跑步机**——追闭源、快速移动的目标(官方已到 0.3.199 补丁位,基本周更),
   永远落后 N 版。
2. **优化错对象**——BPT 的价值是直连引擎 + 无子进程部署 + 摆脱被禁 `claude.exe`,
   不是「表面像官方」。
3. **秘方层 + 主权选择封顶行为保真**——即便默认提示词已忠实开放再现官方主循环,行为保真仍被
   秘方层(公开重构 ≠ 官方线上活体提示词 + 未文档化行为)与模型/provider 主权(换模型换手感)封顶,
   「完美逐比特追平」结构上仍不可得,得认这根本矛盾。

## 4. 采纳的定位:兼容表面、独立引擎、钉基线、选择性追踪

- **追表面、不追行为**——让 BPT 代码换 import 即可切(surface drop-in),但**明确放弃
  行为逐版追平**;兼容性**钉在文档化基线**(「兼容 0.3.x 表面,截至某日」)而非「永远最新」。
- **拥抱对 BPT 有意义的分叉**——直连引擎、provider 配置、自有工具是特性不是缺陷。
  其中 **provider 可替换性升格为定位硬承诺(2026-07-04 拷问修订)**:多 provider
  (含国产模型)是元工具层主权的组成部分,不是备胎配置。诚实代价:可替换 ≠ 等效,
  换模型必然换手感(行为保真度结论在此二次适用)。**以 Claude 体验为范式参考,但不作限定**
  ——0.3.199 表面基线是工程锚,不是体验锚,BPT 的体验主权在自己手里。
- **选择性追踪**——只追 BPT 真用得上的 P0/P1 表面(完成度审计已证 P0 基本持平);
  **故意不追** CLI 耦合长尾(沙箱/设置引擎/CLI 系统提示词/Bedrock·Vertex/OTel/
  WarmQuery/33 变体流)——那些正是 BPT 不需要的 CLI 耦合长尾。

## 5. 稳定性:拆两半,方向相反

- **(a) 代码本身可靠性(会不会崩/挂)** → **持平甚至更强**:更小、全自有、全测试
  (629 测试 + 双驳斥审查),**无子进程 IPC 层**——官方一整类失败模式(子进程 spawn、
  stall 看门狗、stream idle 超时、CLI 版本错配)在我们这儿不存在。
- **(b) 行为相对上游的稳定性** → **随每次上游发版单调恶化**(除非主动追)。这才是
  「始终对照更新」定位的真正稳定性风险:不是代码会坏,而是「与官方一致」这个承诺本身
  不稳定。钉基线正是为消解这条。

## 6. 效率复利:不摊在「每一处」,压在四个齿轮上

「处处低效累积成大差别」是错误模型。复利只发生在能被「轮数」乘数放大的地方,集中在四点:

| 齿轮 | 复利机制 | 现状 |
|------|---------|------|
| **① 提示缓存** | 不缓存则每轮全价重发静态前缀(系统提示+工具);20 轮任务静态前缀差约 7× | **已默认开**(2026-07-04) |
| **② 压缩正确性** | 触发错 → 400 或丢上下文 → 重复劳动 → 多花轮数 | 已修 CJK 低估 + 退化预算 |
| **③ 工具结果 token 纪律** | 截断不精则每条 tool_result 更大 → 下一轮输入更多 | 可调,持续优化 |
| **④ 系统提示驱动的轮数** | 系统提示的残余秘方层差异影响模型规划 → 轮数波动 → 成本+延迟乘数 | 靠 A/B 量化(v5 默认已实测多轮 ~3× 便宜) |

延迟(墙钟)大部分是噪声(被模型推理时间淹没);启动/流式我们反而更快(无子进程、无 IPC)。

**策略:盯死这四把大刀,其余接受它是噪声(80/20)。前两把油已上,第三把持续磨,
第四把靠测量。**

## 7. 测量强制令:把「复利担忧」从未知变数据

复利效率**不能估,得测**。要求:每次 run 埋**预算仪表**(每 run 的 token 分解 /
轮数 / 墙钟 / 缓存命中率 / 每工具调用耗时),在一组**代表性 BPT 任务**上对基线做 **A/B**。
产出一份**按 offender 排序的优化清单**,照头部打。**管不了测不了的东西。**(列入 v0.3。)

## 8. 上行面(不是自我安慰)

因掌控全栈,在 BPT 的**特定工作负载**上我们能比通用官方**更省**:为 BPT 对话模式放
更激进的缓存断点、把压缩专调到**中文**工作负载(官方替你调不了)、砍掉 CLI 全部开销。
**专用且更瘦**是真实上行。
