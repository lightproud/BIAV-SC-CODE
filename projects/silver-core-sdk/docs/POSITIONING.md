## 1. 定位声明

Silver Core SDK(BIAV-SC 引擎)是黑池五层介质架构中的**引擎层**:最慢的介质、
版本化、可替换的商品泵。它的职责由三条否定与一条肯定完整定义:

1. **不认识斜杠。** 引擎不拦截、不解析、不展开任何以 `/` 开头的文本约定。
   用户输入长什么样、如何触发能力,是客户端层的 UX 决策。`query` 收到的 prompt
   一律原样进入装配。
2. **不持有钟。** 引擎不含任何跨 query 的定时、轮询、编排逻辑。钟属于 runner
   守护层(宿主进程)。引擎内的一切时间语义(重试退避、空闲看门狗、工具超时)
   都活在单次 query 生命周期之内。
3. **不追宿主面。** 引擎的宿主侧 API(query 签名、options、编排语义)按黑池
   需求自主演化,不以官方 Claude Code / claude-agent-sdk 的宿主侧形状为义务。
4. **只做一件事:把一次结构化的 agent 调用执行到位,并把模型侧表面保持在模型
   训练分布附近。**

模型侧表面(工具名、参数 schema、系统提示词习语)持续贴近官方形状——受益者是
**模型本身的调用稳定性**,而非替换退路。直连 Messages API 的轻快响应(fetch + SSE,
无子进程、无 IPC、无 CLI 启动开销)是核心价值:**任何提案若代价是引入 CLI 式
中间层开销,直接拒绝。**

### 1.1 扩展面总原则(三种缝)

引擎对外的全部扩展面限定为三种缝:

| 缝 | 是什么 | 例 |
|----|--------|----|
| **钩子** | 引擎定义的时机槽位 | PreToolUse / Stop / PreCompact |
| **工具** | 模型侧能力 | 内建工具 / MCP / 可选导出工具定义 |
| **结构化配置** | 装配期声明 | options 字段 / provider / goal 配置 |

任何新需求先归类为**时机、能力或声明**之一,不引入第四种缝。

### 1.2 钩子契约总则

引擎拥有**「何时」与「契约」**:回调在哪些生命周期节点被调用、可见什么、返回
什么、返回值如何影响引擎行为。宿主拥有**「什么」**:回调内容,引擎不知情也不
关心。

回调的生命周期**不得超出它所嵌的 query**——在钩子内挂墙钟或等待外部世界变化,
视为将 runner 层的钟走私进引擎,**定位违规**。此总则对既有与新增钩子一体适用
(执行细则见 `docs/ARCHITECTURE.md` §E)。

## 2. 兼容政策

1. **不承诺 drop-in 兼容。** 兼容基线不构成待办来源:官方表面出现而本引擎缺席
   的字段/方法/消息变体,本身不构成工单。
2. **COMPAT 文档是参照笔记**(`docs/COMPAT.md`):记录官方做法作为设计输入,
   不是义务清单。
3. **对标 chase 只有两个触发条件**:(a) 新模型发布(模型侧分布可能移动);
   (b) 银芯自行设计新的模型侧表面时(参考官方作业)。此外不设常态追版任务。
4. **主权退路的保障来自引擎自持与测试资产**(全量单测全绿 + 变异测试纪律),
   不来自「可换回官方」。

## 3. 与消费方的关系

- **唯一消费方向**:银芯 → 黑池单向输出(tarball pin 消费);黑池侧 runner
  守护层与客户端层在引擎之上自建钟、UX 与编排,引擎义务是让它们**可被外部
  零补丁地干净构建**(循环支撑接口面见需求档 §3)。
- **版本纪律**:凡改动发货运行时的 merge 必 bump 版本并记 CHANGELOG(随
  tarball 发货),消费方按版本 pin / 回退 / 对账。
- **引擎版本声明面**:运行时可查询的表面版本声明供热更新能力层(技能 md、
  MCP 清单)做加载时兼容校验。

## 4. 工程红线(不因定位改写而变)

- §1.1-HC 黑池防火墙:黑池数据以任何形式不进本仓库;引擎产物单向输出。
- 公开信息纪律:模型侧表面的再现取材自公开发行渠道产物与公开文档(署名保留);
  内部泄漏衍生物一律不读不引。
- 诚实红线:不描述不存在的能力;测试不为转绿而弱化断言;`faithful` 片段
  经 corpus-sync 对账。
