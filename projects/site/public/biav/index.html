Etymology
命名缘由

Brain in a Vat 出自笛卡尔与普特南的同名哲学思想实验：一个脱离躯体、被维持在营养液中的大脑，能否区分"真实经验"与"被注入的经验"？

Studio 选择此名作为身份锚点，并非声称给出答案，而是接受其前提：制作者所感知的世界，本身可能就是被构建的。这个不确定性是创作《忘却前夜》克苏鲁式认知恐惧叙事的精神基底——所有"已知"都可能被更高阶的存在证伪。

在工程层面，本仓库 brain-in-a-vat 就是这个思想实验的具象——一个由 AI 协作维护、自动情报循环驱动的"脱体大脑"，承载 Studio 的记忆、决策与协议。

Silver Core
银芯系统

银芯（Silver Core / BIAV-SC）是 BIAV 项目的知识层系统名，定位为 Studio 的公开信息层（整层公开，自身工程产物亦属公开信息）：负责采集、整理外部社区与解包信息，向内部层（黑池）单向输出，是 Studio 对外的"眼睛和耳朵"。需注意：定位公开不解除第三方平台对采集的约束，黑池→银芯方向亦永久关闭——越公开，这道单向防火墙越关键。

银芯的记忆不再依赖自建检索栈，而是收回到三条互补支柱：CLAUDE.md（每会话自动加载的统一入口与协议）、memory/*.md（人工策展的决策 / 踩坑 / 状态 / 方法论档案），以及 Claude 平台原生的上下文管理。每一轮协作产出的事实与决策被写入策展档案，让下一轮会话从已有沉淀起步。

会话之外，MCP 服务 biav-sc-memory 提供四件协作工具：character_persona（人格激活）、record_decision（决策落档）、record_lesson（教训沉淀）、current_continuity（连续性读取）。这是 Studio 在 AI 协作时代的方法论尝试：让记忆成为基础设施，让协议优先于工具。
