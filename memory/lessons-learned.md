- **坑**：全局 sed 删标签致 div 失衡，PDF 渲染异常。

## 42. 两条自动化在文件系统上对冲成永动机

- **坑**：归档 CI 删数据、回填 CI 每小时写回来，「删→写回」无限循环 2.65GB；`[skip ci]` 互不触发 ≠ 互不冲突。
- **准则**：同路径写盘的多条自动化必须共享「谁该留谁该删」的单一事实源；一次性回填任务必有显式「完成」终态，否则「指针为空就重启」变永动机。
- **案卷**：lessons-archive.md 案卷 #42。

## 43. 否定性/穷尽性结论须穷尽侦察（并入 #35/#36/#37/#40/#41）

## 44. A/B 候选臂静默失效，「无差异」是假象

- **坑**：变量只在某配置路径生效，harness 没走那条路径，两臂实际同配——测出的「无差异」是管道 bug 不是结论；修复后结论整个翻案。
- **准则**：测量前先断言两臂实际入参有别；「仅特定条件下生效」的开关，测量必显式满足该条件并测试锁定；量错了公开翻案留痕，不静默改数。
- **案卷**：lessons-archive.md 案卷 #44。

## 45. rebase 冲突里 --ours/--theirs 语义反转

- **坑**：rebase 时 HEAD 是基底，`--ours` = main 侧——「取我方」拿到对方版本，且双源互证守卫不防「一致地错」。
- **准则**：冲突处置不按方位词挑，先认清谁是 HEAD 或按冲突块语义挑；`rebase --continue` 前对冲突文件 grep 关键值核对；sed 替换后必回读命中（静默零命中是漂移温床）。
- **案卷**：lessons-archive.md 案卷 #45。

## 49. CCR 例程事故簇：钩子误报 / fresh-session 空环境 / bot 身份污染（并入 #47/#48）

- **坑**：CI 一次性容器惯用写法进长命自绑定容器全翻车——仓库级 `git config user.*` 跨进程污染人工会话提交、平台钩子按陈旧远端指针误报「需改写上游提交」、fresh-session 不继承仓库源。
- **准则**：长命/共享容器脚本禁写仓库级持久配置，一律 `git -c` 单命令作用域（从 CI 抄脚本必过此检查）；例程需仓库+凭证选自绑定、独立任务才 fresh-session、提示词写中性事实；自动化要求改写非本会话签发的提交时，先 `git rev-parse HEAD origin/main` 对照归属再动手。
- **案卷**：lessons-archive.md 案卷 #49。

## 50. unref 计时器用在进行中的重试退避 = 无头进程静默死亡

- **坑**：退避计时器 unref 后常是最后存活句柄，事件环排空、纯脚本消费方 exit 13；vitest 宿主句柄常驻测不出此病。
- **准则**：写 `unref()` 前先答「计时器醒来时是否还有别的句柄必然存活」，答不上就别 unref（正当对象仅空闲看门狗/空闲池化 socket）；「进程能否活过恢复窗口」只能真子进程测，测试宿主内绿灯不作数。
- **防护**：`tests/replay-backoff-process-exit.test.ts`（子进程级回归）。案卷：lessons-archive.md 案卷 #50。

## 51. 机械结论先无钥证，判卷预算只买主观质量分

- **坑**：环三首轮把「引擎有没有做某动作」这类**纯机械**发现（dc-05 台账双桶、dc-03 续写旗）也送进付费 LIVE 判卷轮打分，烧了本可省的 judge 预算；充值受阻后才回头用无钥模拟器一断言证清白（引擎无罪）。判卷官（真 Sonnet 5）测的是**主观质量**，模拟器测的是**代码机器**，二者不可互替——但机械项误用了贵的那把尺。
- **准则**：一条评估发现动手前先分诊——「答案能否用确定性断言判真伪」？能（台账/计数器/是否重驱动/是否走某分支）→ **先无钥模拟器**（`tests/eval-harness-faults.test.ts` / `emulator-e2e.test.ts` 模式，零 API）；只有「答得好不好」这类真·主观分才花判卷钱。付费买的是模拟器给不了的真模型判断 + 真端点互通，不是机械回归。
- **防护**：`tests/eval-harness-faults.test.ts`「dc-05 mixed-fault ledger」为机械项无钥范例；升格候选 = runEvals 出题期给每题标 `assert_kind: mechanical|subjective`，机械项禁入判卷轮（待守密人裁）。案卷暂留本档。
