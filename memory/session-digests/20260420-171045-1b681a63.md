## 三、认知教学：本会话的运作原理（守密人需理解的四个概念）

**概念 1：主控台不写代码**
艾瑞卡是战略锚点 + 知识搬运工。写代码会**稀释上下文**（Claude Code 的上下文窗口有限，一旦塞满会开始压缩丢失早期记忆）。主控台要活 90 天，就必须保持瘦身。

**概念 2：子代理是"一次性臂"**
每次派发 `Explore` / `general-purpose` 子代理，等于开一条独立上下文跑任务，只把**摘要**带回主控台。主控台累积"摘要"而不是"原始材料"，这是长寿命的关键。

**概念 3：新会话是"执行终端"**
当某任务需要反复编辑代码（比如 Wiki 数据补全）时，应该**开新会话**（Code-wiki），由主控台派发任务 + 明确产出要求。新会话做完合并到 main 后关闭，主控台只读其产出的 commit 摘要与 memory 档案更新。

**概念 4：记忆档案是跨会话"共享硬盘"**
`memory/decisions.md`、`memory/project-status.md`、各 `CONTEXT.md`——这些是**档案层持久记忆**，比任何单一会话的上下文更长寿。任何战略决策必须写进档案，否则 30 天后回看就是空气。
