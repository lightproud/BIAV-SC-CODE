## Chunking（切片 / 分块）是什么

**问题**：你有一个 1-2MB 的 JSON 配置文件，里面装了一万个角色的属性数据。用户问"哪里控制了狂战士的暴击率"。BPE 怎么帮他找？

**笨办法**：把整个 1-2MB 文件当作一个整体，做个"语义指纹"。结果：这个指纹代表"所有一万个角色的平均语义"，用户搜什么都能匹配上，但也就**什么都搜不到重点**——因为"狂战士"这一行被其他 9999 行稀释掉了。

**正确办法**：**在建索引的时候，先把文件切开**。切成一万个小块，每块就是一个角色。然后每个小块单独做"语义指纹"。用户搜"狂战士暴击率"的时候，能精准找到狂战士那一行。

**这就叫 chunking**。关键是**按语义切**，不是按长度切：

| 文件类型 | 怎么切 |
|---|---|
| CSV | 一行一个切片（每行是一个角色/道具/技能） |
| JSON / Lua 表 | 一个顶层条目一个切片（一个角色对象 = 一个切片） |
| 代码 | 一个函数 / 一个类 = 一个切片 |
| 长文档 | 一段 / 一节一个切片 |

**对你的具体情况**：你说 CSV 是策划源，然后转 JSON 和 Lua。那 chunking 的情况非常理想——**因为这些配置本来就是结构化的**。一个 JSON 顶层 key、一个 Lua 表条目、一行 CSV——天然就是一个切片边界。这比那种"一堆散文字"的 chunking 简单得多。我们能写出很精准的切片器。

**1-2MB 的配置会被切成多少块？** 假设每块 1KB（一个典型角色条目），1.5MB ÷ 1KB ≈ 1500 块。120 个配置按平均 500 块算，总共约 6 万个切片。再加代码（一百万行，按每 20 行一个函数算 5 万个函数）——**整个黑池的索引规模大约 10-15 万条目**。这个量级对 SQLite+向量检索非常轻松，完全不是性能瓶颈。

## Embedding 和 Embedding API 是什么

**核心操作**：把一段文字变成一串数字（一个向量）。

比如把"狂战士 HP=2000 暴击率=15%"这一段文字，喂给一个叫 embedding model 的神经网络，它吐出来 600 个浮点数，像这样：

```
[0.031, -0.128, 0.876, 0.042, ..., -0.201]  ← 600 个数
```

这串数字**代表这段文字的语义**。魔法在于：**语义相似的文字，吐出来的向量在几何上也相近**。

所以用户问"哪里控制了狂战士暴击率"时，BPE 会：
1. 把用户这句问句也丢给 embedding model，拿到一个 600 维向量
2. 拿这个向量去和之前存好的 6 万个切片向量做距离计算
3. 距离最近的 top-20 就是候选答案

**这是语义搜索的全部秘密**——不是 grep，不是关键词匹配，而是"向量几何相似度"。它能搞定"哪里控制了某个行为"这种自然语言查询，就是因为它不依赖关键词重合。

### 那 "embedding API" 是什么

做 embedding 需要一个专门训练的神经网络（embedding model），它和 Claude、GPT 这种对话型 LLM 是两种不同的模型。**让 BPT 拿到 embedding 有两条路**：

**路径 A：调远程 embedding API（类似调 Claude API）**
- BPT 在建索引的时候，把每个切片的文字发给一个远程服务
- 服务返回对应的向量
- 典型代表：OpenAI 的 `text-embedding-3-small`、智源的 BGE API、阿里云的通义 embedding
- **好处**：BPT 自己不用带模型，代码轻
- **坏处**：每一条切片都要走一次网络请求；而且**你的公司网关得提供这个服务**才行

**路径 B：本地跑一个小的 embedding 模型**
- BPT 自己带一个 ~400MB 的小神经网络文件（比如 `bge-small-zh-v1.5`）
- Python 端把它加载到内存里，切片文字丢进去，直接出向量
- **好处**：不依赖网关、不耗网关额度、断网也能建索引
- **坏处**：要往 SVN 塞 400MB 的模型文件；需要 Python 环境能跑 sentence-transformers（CPU 就行，不一定要 GPU）

**Claude / Anthropic 官方**目前**没有提供 embedding API**——Anthropic 自己不做 embedding 模型，他们官方推荐用 Voyage AI。所以 "我们整体都用 Claude" 这个偏好**天然不能覆盖 embedding 这一步**，我们必须单独决定。
