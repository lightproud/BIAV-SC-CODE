**关键点：这是 AI 的"记忆力"。** 没有它，BPT 就是个普通 Claude 对话窗口；有了它，AI 能查你们项目的历史决策、角色关系、知识图谱。

---

### 2. Direct 直调 (`direct-client.ts`)

**作用：UI 和系统后台直接调 Python 脚本，不经过 LLM。**

工作方式：
- 每次调用 `spawn` 一个**一次性 Python 进程**，跑完就退出
- 捕获 stdout 作为 JSON 结果返回

暴露的 **5 个工具**（不进 LLM schema，零 token 消耗）：

| 工具 | 谁调 | 干什么 |
|------|------|--------|
| `checkCache` | 系统启动时 | 检查做梦 Agent 的预计算缓存 |
| `memoryUtility` | 银芯面板 UI | 查看文件效用排名（哪些文件最常被引用） |
| `recommendContext` | 系统自动/面板 | 按当前问题推荐该读哪些文件 |
| `rebuildIndexes` | 管理按钮 | 重建银芯全部搜索索引 |
| `memoryWriteback` | 管理按钮 | 把 git 变更提取为知识写入图谱 |

**关键点：这些是管理/维护操作。** 让 LLM 调 `rebuildIndexes` 既浪费 token（schema 常驻）又危险（LLM 不该决定何时重建索引）。所以拆出来走直调。

---

### 为什么说可以独立部署

这两处都依赖 `scripts/` 目录下的 Python 脚本。但代码里已经有容错：

- MCP 连不上 → 状态栏显示 "disconnected"，对话功能正常，只是 AI 没有记忆力
- Direct 调用失败 → 银芯面板显示错误，不影响其他功能

所以 BPT **本体不依赖银芯就能跑**——纯 LLM 对话 + BPE 黑池搜索完全独立。银芯是增强层，按方案 A 做成可选配置项就行。
