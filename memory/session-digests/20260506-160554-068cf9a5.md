## 三、五个可能改进方向（架构选择，需守密人裁定）

### 方向 A — **事实卡片化**（Fact Card）

把碎片信息提炼成原子化「事实点」(facts.json 已有雏形，扩展之)。

```json
{"id": "fact_001", "subject": "钥令", "predicate": "充能资源", "object": "钥焰", "evidence": "morimens-context.md:133", "topic": ["卡牌系统", "v2.0"]}
```

| 优势 | 劣势 |
|---|---|
| 原子化、可关系化、易去重 | 信息浓缩可能丢失叙事上下文；维护成本高 |
| AI 检索拿到「事实」而非「片段」 | 需要工具辅助提取（半自动化） |

**适合**：游戏机制、决策记录、设计原则、术语定义

### 方向 B — **主题入口卡 + 大纲**（Topic Outline）

每个主题建一份「入口 md」（≤ 100 行），含：
- 一句话摘要
- 关键事实 5-10 条（链接到详细文件）
- 相关主题（横向关联）
- 历史变化（时间线）

```
memory/topics/
├── README.md                     # 全主题地图
├── strategic-plan.md             # 入口卡 → 链接 5 份散落档案
├── memory-system.md              # 入口卡 → 链接 4 份散落档案
├── card-system.md                # 入口卡 → 链接 morimens-context + card-system.json + lua
├── ...
```

| 优势 | 劣势 |
|---|---|
| AI 先读入口卡（100 行）建立全局认知，再深读 | 需要持续维护——主档案改了入口卡也要改 |
| 解决「同一主题 N 份文件」问题 | 入口卡可能重复内容 |

**适合**：所有现有散落主题。可由 dream.py 半自动生成。

### 方向 C — **问答对索引**（Q&A Pair Index）

把所有信息预生成「常见问题 + 权威答案 + 引用」三元组。

```yaml
- q: "钥令是什么？"
  a: "v2.0 引入的守密人技能系统，由钥焰充能至 1000 释放。"
  refs: [morimens-context.md:93, card-system.json#keeper_skill_system]
```

| 优势 | 劣势 |
|---|---|
| 直接命中查询意图（hot-path 优化） | 未预设的问题无法答 |
| 检索语义匹配 question 而非 chunk | 需要预先列出可能问题（覆盖率难） |

**适合**：高频常见问题的快速通道（与方向 A/B 互补）

### 方向 D — **分层记忆**（Tiered Memory，类比 LLM context）

按 LLM 上下文窗口物理特性显式分 4 层：

```
L1 — 启动包（< 5 KB）：boot-snapshot.md（已有）
L2 — 主题入口（每主题 < 5 KB）：方向 B
L3 — 详细档案（每档案 5-50 KB）：当前 memory/*.md
L4 — 历史档案（无限）：session-digests/ + archive/
```

AI 按需逐层下钻：先读 L1 → 命题相关时拉 L2 → 需要细节时拉 L3。

| 优势 | 劣势 |
|---|---|
| 与 LLM context window 物理特性匹配 | 需要重组现存所有档案 |
| 加载策略明确（context_manager 4 层推荐已有雏形） | L2 层需要新建 30+ 入口卡 |

**适合**：根本性重构，与方向 B 配套

### 方向 E — **叙事性记忆**（Narrative Memory）

按时间线 / 决策链 / 演化轨迹组织，AI 拿到的是「连贯的故事」。

```
memory/timeline/
├── 2025-q4.md    # 项目立项 + 三轴定义
├── 2026-q1.md    # 三轴落地 + 黑池战略
├── 2026-04-19.md # BPT 战略转向（关键转折点）
├── 2026-04-26.md # v2.0 三新使命
```

| 优势 | 劣势 |
|---|---|
| 因果链清晰，AI 理解「为什么」 | 组织成本最高 |
| 适合战略类 / 决策类知识 | 不适合静态游戏数据 |

**适合**：决策、战略、阶段演化
