# Startup Hook Skill for Claude Code on the web

Create SessionStart hooks that install dependencies so tests and linters work in Claude Code on the web sessions.

## Hook Basics

### Input (via stdin)
```json
{
  "session_id": "abc123",
  "source": "startup|resume|clear|compact",
  "transcript_path": "/path/to/transcript.jsonl",
  "permission_mode": "default",
  "hook_event_name": "SessionStart",
  "cwd": "/workspace/repo"
}
```

### Async Mode
```bash
#!/bin/bash
set -euo pipefail

echo '{"async": true, "asyncTimeout": 300000}'

npm install
```

The hook runs in background while the session starts. Using async mode reduces latency, but introduces a race condition where the agent loop might depend on something that is being done in the startup hook before it completed.

### Environment Variables

Available environment variables:
- `$CLAUDE_PROJECT_DIR` - Repository root path
- `$CLAUDE_ENV_FILE` - Path to write environment variables
- `$CLAUDE_CODE_REMOTE` - If running in a remote environment (i.e. Claude code on the web)

Use `$CLAUDE_ENV_FILE` to persist variables for the session:
```bash
echo 'export PYTHONPATH="."' >> "$CLAUDE_ENV_FILE"
```

Use `$CLAUDE_CODE_REMOTE` to only run a script in a remote env:
```bash
if [ "${CLAUDE_CODE_REMOTE:-}" != "true" ]; then
  exit 0
fi
```

## Workflow

Make a todo list for all the tasks in this workflow and work on them one after another

### 1. Analyze Dependencies

Find dependency manifests and analyze them. Examples:
- `package.json` / `package-lock.json` → npm
- `pyproject.toml` / `requirements.txt` → pip/Poetry
- `Cargo.toml` → cargo
- `go.mod` → go

## 5 个依赖的角色架构（小学生版）

news 聚合器其实是**三段式兜底**架构：


每一层各司其职，**互相不重叠**，但**有可能砍掉中间层**。

## 合并选项分析

### 选项 A：删 `cloudscraper`，让 `playwright` 兜底 Cloudflare 站

**可行性**：✅ 技术上可行
- cloudscraper 当前只用在 NGA 等少数 Cloudflare 站
- playwright 浏览器天然能绕 Cloudflare（真实 Chrome 指纹）
- 代码改动：把 `_get_cf()` 函数内部从 cloudscraper 改成 playwright

**代价**：
- 抓 NGA 等站从**毫秒级**变成**秒级**（启动浏览器实例）
- 每天聚合总时长增加几分钟
- workflow 跑得更慢

**收益**：少 1 个依赖（5 → 4）

**评价**：⚠️ 性能损失明显，依赖减少不痛不痒

---

### 选项 B：删 `requests`，全用 `cloudscraper`

**可行性**：✅ API 兼容（cloudscraper 内部就是 requests）
- 所有 `.get()`/`.post()` 调用语法完全一样
- 只是把 import 换一下

**代价**：
- 每次请求多一层 Cloudflare 检测开销
- cloudscraper 维护没 requests 活跃，潜在 bug 风险
- 升级路径变窄（依赖一个小众库）

**收益**：少 1 个依赖（5 → 4）

**评价**：⚠️ 潜在稳定性损失，不推荐

---

### 选项 C：删 `google-play-scraper`，自己用 `requests` 调 Google Play API

**可行性**：✗ 风险大
- Google Play 的 API 是**非公开**的，google-play-scraper 是逆向出来的
- 自己实现等于重新逆向，且需要持续维护（Google 改 API 就崩）

**评价**：✗ 不推荐

---

### 选项 D：删 `anthropic`，放弃 Discord 月度 AI 总结

**可行性**：✅ 简单删除即可
- 只在 `discord_archiver.py` 里月度调用
- 删了月报变成「光归档没总结」

**评价**：✗ 功能性削减，不算「合并」

### 选项 E：极端方案——只保留 `playwright`

**可行性**：技术上行，但**疯狂**
- playwright 启浏览器抓任何东西
