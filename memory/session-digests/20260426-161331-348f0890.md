## 6. 风险清单（参考 lessons-learned.md）

1. **批量生成必跑 build**（lesson #15）：`generate_pages.py` 跑完 72 角色 + N trinkets 后，**必须**在提交前本地 `npm run build`。缓解：在 generate_pages.py 末尾加 `--verify` 选项，自动调用 vitepress build 子进程
2. **frontmatter 含冒号必须加引号**（lesson #12）：角色名 / 神器名 / 卡池标题任何含 `:` 的字段写入 frontmatter 都要 `"..."` 包裹。缓解：`generate_pages.py:108-113` 已有逻辑，扩展到 trinket/banner 生成器
3. **img src 以 `/` 开头会被 Vue 当 import**（lesson #13）：所有 portrait/CG 引用走 `:src="'/path'"`。缓解：在 PortraitGallery / TrinketCard 等组件里强制走动态绑定，不允许 raw `<img src="/...">`
4. **sed 批量替换破坏结构**（lesson #1）：72 角色页的批量调整禁用 sed，统一通过重跑 `generate_pages.py` 完成。缓解：把"页面是脚本产物"作为铁律写进 `projects/wiki/CONTEXT.md`
5. **CONTEXT.md 与实际状态脱节 + 档案声明 vs 实际文件交叉校验**（lesson #4 / #25）：每个 Phase 完成后必须更新 `projects/wiki/CONTEXT.md` 的"当前状态"段，并附 `[last-verified: YYYY-MM-DD]`
6. **多会话格式冲突**（lesson #21）：若 Code-wiki 与 Code-主控台同时改 `characters.json`，按字段级合并而非整体覆盖。缓解：Phase 0 引入 `scripts/merge_character_data.py` 做字段级合并
7. **大文档 Write 超时**（lesson #26）：本次蓝图涉及多份新 schema（每份 200-400 行），子代理产出时**必须分段**——先 Write 骨架 < 200 行，再 Edit 追加
8. **主控台越界写业务代码**（lesson #27）：本计划全程由 Code-wiki 会话执行，主控台仅派发与转交产出，不亲自落盘 schema 或运行 generate_pages

  - `recommended_for`: array of character.id, optional
  - `lore`: string, optional
  - `icon_url`: string nullable, required
  - `last_verified`: date string, required

  - `id`: integer, required

    - `sr_base`: number, required
    - `sr_combined`: number, required
    - `r_base`: number, required

  - `pity`: object, optional

  - `start_at`: date string, optional
  - `end_at`: date string, optional
  - `version`: string, optional（首发版本）

- `description`: string, required
- `groups`: array, required
  - `id`: integer, required
  - `name`: string, required
  - `desc`: string, required
  - `type`: string, required（幻梦深潜 / 怪雾之都 / 混沌之域 等）

  - `stages`: array of stage_id, required（外键到 stages 数组）

  - `first_clear_rewards`: array, optional

  - `id`: string, required, pattern `^[a-z0-9-]+$`
  - `name`: string, required
  - `name_en` / `name_ja`: string, optional
  - `category`: string, required（外键到 categories）
  - `rarity`: enum, optional
  - `description`: string, required
  - `usage`: string, optional（用途说明）
  - `obtain_sources`: array, optional（卡池 / 关卡 / 兑换 / 任务）
