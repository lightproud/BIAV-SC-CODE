- **AI Agent 搜索基础设施**(`anysearch-ai`,github.com)——守密人指的几乎肯定是这个。
- 另有 Chrome/Safari 的「右键搜索」扩展、macOS 本地文件搜索 app,以及一个**同名 Mac 浏览器劫持广告软件「AnySearchManager」**(恶意,与前者无关)。命名撞车,留意别混淆。

## 它是什么(AI Agent 那个)

一个面向 AI Agent 的**统一实时搜索引擎**,开源在 GitHub,两种形态:

- `anysearch-skill`——下载解压丢进 `~/.claude/skills/anysearch` 即可,Claude Code / Cursor / Windsurf 通用
- `anysearch-mcp-server`——走 Streamable HTTP 的 MCP server(MCP spec 2025-03-26)

能力:通用网页搜索 / 垂直领域搜索(金融、法律、学术、安全、代码等)/ 单次并行 5 路查询 / URL 全文抽取转 Markdown。API key 可选(匿名也能用,限速更低),免费 key 在官网控制台领。许可证与隐私条款 README 里没写清——这是个待核实点。

## 对银芯有没有帮助

判断:**是个正经工具,但它解决的不是银芯当前的检索痛点。** 理由分三层——

1. **和 `memory_search.py` 不是一回事**。本仓库的检索痛点(上一轮基线那份)是**内部 RAG**:排序被会话日志噪音干扰、设定数据本身缺失。AnySearch 搜的是**公开互联网**,既不索引本仓库,也变不出仓库里没有的卡牌/界域设定。这两个缺口它一个都补不了。

2. **和 news 采集管线职能重叠有限**。使命1的采集层是**平台专用**的(Discord 走 bot token 归档、Steam/Bilibili/NGA 走各自接口),AnySearch 是通用网页聚合,替代不了这些定向管线。

3. **真正可能有用的位置**:做梦 Agent(深睡趋势分析 / REM 周报)目前没有实时联网搜索能力,给它接一个 MCP 网页搜索做事实落地(grounding),是合理的增量——这是 AnySearch 唯一对得上银芯的场景。

## 推荐

不建议当「银芯检索升级」来上——那是个误配。**若守密人想给做梦 Agent / 临时外部调研加一条联网搜索能力**,它作为 MCP server 形态值得一试,成本低(匿名可跑)。上之前要核实两件事:许可证、以及外部服务的隐私边界(银芯是公开层,查公开信息无碍,但查询内容会出网)。

要不要艾瑞卡帮守密人做件具体的:把 `anysearch-mcp-server` 接入做梦 Agent 的可行性验证一轮(查清许可证 + 跑一个真实查询测延迟与召回),再决定要不要落盘?
