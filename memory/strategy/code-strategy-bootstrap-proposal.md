## 二、议题三选

### ◇ 议题 A：忘却前夜社区温度长期评估框架 + 沙耶联动实战观察

**一句概括**：建立「社区温度长期评估」结构化方法，并以即将到来的沙耶之歌联动作为第一次基线测量与压力测试。

**与三新使命的关联**：使命 #1「黑池公开信息入口」。当前日报系统采集了 24 平台原始数据，但缺乏长期温度评估的结构化输出——黑池消费侧拿到的还是原始流，没有「本周社区情绪走势」「关键事件影响力衰减曲线」「头部 UGC 创作者画像」等可消费的高阶分析。联动是天然的检验窗口：联动公布前 / 公布中 / 上线后 / 上线 2 周后四个时点，对比社区情绪与平台分布变化，能直接验证「黑池的眼睛和耳朵」是否真的看见了关键信号。

**预期产出形态**：
1. `memory/strategy/community-temperature-framework.md`（长期文档）：温度评估方法论 + 关键指标（情绪指数 / 头部 UGC 标记规则 / 跨平台传染性 / 沉默平台告警）
2. `memory/research/saya-collab-baseline-2026-04.md`（一次性档案）：联动公布前基线读数 + 第一轮观察记录
3. 与 Code-news 的接口建议（提议主控台派发，本会话不直接派发）

**预计深度对话轮次**：3 ~ 4 轮
- 轮 1：方法论框架草案 + 守密人确认指标维度
- 轮 2：第一次基线读数（联动公布前）
- 轮 3：联动公布后第一周观察 + 框架校准
- 轮 4（可选）：联动 2 周后衰减曲线 + 框架定版

**风险**：联动公布日期未确定，若窗口开在 Phase 2 后段，框架价值要等较久才能验证。可降级方案——先用 v2.4.3 莫丝新唤醒体上线作为预演事件。

---

### ◇ 议题 B：使命 #2「社区共建知识底座」真实贡献者获客路径评估

**一句概括**：在 Fandom / Bilibili Wiki / NGA 等已存在多年的竞品环境下，银芯 Wiki 凭什么吸引外部贡献者？做竞品分析 + 贡献者画像 + 价值主张选项推演。

**与三新使命的关联**：使命 #2「社区共建知识底座」。Phase 2 M3 验收硬指标「至少 1 种贡献流程跑通 1 轮」是当前最有不确定性的验收点。`contribution-protocol.md` v1.0 已开通 C3（社区贡献者）fork + PR 通道，但流程通了不等于有人贡献。如果价值主张不清晰，外部贡献者会继续去 Fandom（已经有词条结构 + 流量 + 编辑社群），M3 验收就会卡死。

**预期产出形态**：
1. `memory/research/community-wiki-competitive-analysis-2026-04.md`：Fandom morimens.fandom / Bilibili Wiki / NGA 资料区 / Discord 维护团 四方对比，覆盖词条数量 / 数据深度 / 三语支持 / 编辑活跃度 / 流量预估
2. `memory/strategy/contributor-acquisition-paths.md`：3 ~ 5 个差异化获客路径选项（如「制作人背书的官方权威定位」「数据源开放可二次开发的 fact-bible 定位」「全语言 Wiki 的国际化定位」「社区研究者的可引用学术定位」），每个含价值主张 / 目标贡献者画像 / 启动成本 / 风险
3. 推荐 1 ~ 2 条主推路径供守密人裁决

**预计深度对话轮次**：3 ~ 5 轮
- 轮 1：竞品扫描 + 守密人确认调研范围
- 轮 2：贡献者画像访谈框架（必要时建议主控台派 Code-news 拉真实数据）
- 轮 3：价值主张选项分析
- 轮 4：守密人裁决主推路径
- 轮 5（可选）：路径 1.0 落地节奏建议

**风险**：竞品流量数据需要外部工具（SimilarWeb / Fandom 公开数据），若全靠采访守密人则评估深度有限。可叠加 Code-news 协助。

### ◇ 议题 C：使命 #3「Studio 团队 AI 协作训练场」具体运作机制设计

**一句概括**：把「至少 1 名 Studio 团队成员（守密人之外）基于银芯做过 1 个公开项目或企划」从模糊目标落成可推进的运作机制——onboarding 包 / 内部分享 / 文档改写 / 试点候选。

**与三新使命的关联**：使命 #3「Studio 团队 AI 协作训练场」。这是三使命中**最模糊**的一个：使命 #1 / #2 都有明确的子项目（news / wiki）作为载体，#3 没有。Phase 2 M3 验收要求「至少 1 名 Studio 团队成员独立产出」，但银芯目前所有文档（CLAUDE.md / BIAV-SC.md / 各 dispatch brief）都是为守密人 + AI 会话写的，**没有面向「不熟悉银芯但要用 AI 做事的 Studio 同事」的入口**。如果 6 月底前不解决，M3 验收会临时抱佛脚。

**预期产出形态**：
1. `memory/strategy/training-field-operation-options.md`：3 ~ 5 个运作机制选项分析（如「onboarding 包：非守密人也能 30 分钟启动一个 AI 项目」「内部分享会 3 ~ 5 场：守密人现身说法演示」「文档分层：把现有文档拆出『非银芯专家入门版』」「试点项目：选 1 个 Studio 同事的真实需求陪跑一遍」），每个含适配人群 / 启动成本 / 守密人时间消耗 / 复制可行性
2. `memory/research/studio-member-receptivity-survey-design.md`：如何在不打扰团队的前提下评估接受度（设计 5 ~ 7 问轻量调查给守密人参考）
3. 推荐路径供守密人裁决

**预计深度对话轮次**：3 ~ 4 轮
- 轮 1：现状评估 + 守密人确认 Studio 团队 AI 使用现状（不公开提名具体成员）
- 轮 2：运作机制选项分析
- 轮 3：守密人裁决推荐路径
- 轮 4（可选）：路径落地节奏建议

**风险**：本议题高度依赖守密人对 Studio 团队成员状态的私下评估（隐私性强），艾瑞卡只能给框架，不能猜成员状态。

## 四、艾瑞卡的偏好（仅供守密人参考）

如果只能挑一个，艾瑞卡建议**议题 A（社区温度长期评估 + 沙耶联动观察）**。

理由：
1. **窗口期最敏感**——联动相关 UGC 已经在 4-26 / 4-27 出现头部 engagement，再不启动框架基线就会丢失「公布前」对照组
2. **直接服务使命 #1 而 #1 是黑池消费侧最先要用的**——黑池建成后「眼睛和耳朵」如果只能给原始流，价值会打折
3. **有可量化产出**——情绪指数 / 头部 UGC 衰减曲线都是可验证的数字，便于 Phase 2 验收时拿出来
4. 议题 B / C 重要但不紧急，可以排在 5 月或 6 月——B 等 contribution-protocol v1.0 跑一轮看真实摩擦点再做更精准；C 等守密人对 Studio 团队 AI 现状有更多观察后再做更扎实

如果守密人偏好议题 B 或 C，艾瑞卡同样可立即展开——三个议题都已完成可行性预判。
