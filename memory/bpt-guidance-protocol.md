## 一、协议目的

银芯不再在仓库内承载 BPT 代码，但 BPT 的开发仍需要银芯提供的能力：战略视角、架构经验、方法论、事实圣经、踩坑记录。

本协议定义**银芯 ↔ 守密人 ↔ BPT** 三方之间的信息流动规则，确保在没有自动化工具链的前提下，指导关系仍然可持续、可追溯、可沉淀。

## 二、角色与职责

| 角色 | 实体 | 职责 |
|------|------|------|
| **指导者** | 任意银芯会话（v0.3：主控台/Code-BPT 专岗随多会话架构 2026-06 退役，指导职能按需承担） | BPT 设计 / 架构 / 方法论议题的对话指导；沉淀 lessons/decisions 写回 `memory/`；不直接访问 BPT 仓库 |
| **裁定者 / 回流过滤器** | 守密人 | 战略裁定 + 黑池现实经亲述认可回流银芯（v0.3 改写，原「搬运者/学习者」）；是唯一跨越银芯/黑池边界的实体 |
| **消费者 / 执行者** | 黑池会话（BPT 侧独立 harness） | 经 fetch 直接消费银芯仓库（通道①）；接收守密人裁定后在黑池内执行 |
| **记忆层** | 银芯 `memory/` 档案 | 跨越会话的持久记忆；黑池可直接 fetch 引用 |

## 三、信息流向（v0.3：双通道，单向可追溯）

```
  ┌─ 通道①（主）：黑池 fetch 银芯仓库 ──────────────────────┐
  │   黑池会话 ──(主动拉取：方法论档案 / 工程产物 / OKF)──┐  │
  │   银芯仓库（公开层，被动被拉取，不知晓也不追踪）  <───┘  │
  └──────────────────────────────────────────────────────────┘
  ┌─ 通道②（对话搬运，收窄为两职能）─────────────────────────┐
  │   a. 战略裁定：银芯会话 ──(拷问/方案/搬运包)──> 守密人 ──> 裁定 │
  │   b. 回流过滤：BPT 现实 ──(守密人亲述认可)──> 银芯档案沉淀     │
  └──────────────────────────────────────────────────────────┘
```

**通道①（主消费通道）**：黑池已 fetch 银芯仓库。方法论（`memory/*` 档案、OKF 思想）与
工程产物（`bpt-agent-sdk`、`kb_navigator`/`build_kb_index` 一族）均由黑池会话直接读取，
不再需要逐条手工搬运。方向合规：fetch 是黑池对银芯**公开层**的主动拉取，数据流仍为
银芯 → 黑池单向，防火墙（§1.1-HC）无涉。**连带纪律**：银芯的档案质量、代码质量、
OKF 结构即接口质量本身——银芯写下的每一份东西，读者名单里都有黑池的 agent。

**通道②（对话搬运，不可替代的残余职能）**：
- **a. 战略裁定**：架构决策、定位议题、需写入 `decisions.md` 的事项，仍经守密人对话裁定。
- **b. 回流过滤**：黑池什么都能从银芯拉，但**银芯对黑池的一切认知只能经守密人过滤进来**
  （亲述认可形式，如「团队已在用 BPT」）。这是防火墙在信息回流方向的唯一合法阀门，
  无可替代、必须明文守住。

**守密人角色（v0.3 改写）**：由「唯一搬运者 + 学习者」改写为「**唯一裁定者 + 回流过滤器**」。
学习价值仍存在，但不再是协议存在的核心理由（BPT 已是团队在产工具，非学习载体）。

**输出形态（2026-07-04 Q6 裁定：A+B 双输出）**：方法论输出（OKF 本体思想、4R 归位纪律、
放指针不放本体、确定性索引家族、踩坑记录）+ 工程产物输出（`bpt-agent-sdk` 先例、
kb 工具族）。两者均经通道①交付。

## 四、对话搬运的标准格式

### 4.1 银芯 → BPT 的指导包

每次守密人要从银芯向 BPT 搬运指导时，银芯应输出一个**可直接复制的搬运包**，包含以下字段：

- **指导主题**：一句话概括
- **背景上下文**：2-3 句话，BPT 会话可能缺乏的背景
- **具体建议**：方案 / 步骤 / 陷阱提示
- **引用档案**：银芯 `memory/` 中相关文件的路径与摘要片段
- **验收问题**：让守密人带回银芯的反馈问题（1-3 个）

### 4.2 BPT → 银芯 的反馈包

守密人从 BPT 会话带回的反馈应包含：

- **指导主题回执**（对应银芯上次的搬运包主题）
- **BPT 实际执行摘要**：做了什么、产出是什么
- **偏差或阻塞**：BPT 没有按原建议执行的地方、遇到的问题
- **守密人的学习记录**：从这一轮中理解到的概念（供银芯沉淀到 `lessons-learned.md`）

## 五、沉淀机制

本协议下产生的"银芯指导 BPT 经验"需在以下文件中积累：

- `memory/lessons-learned.md` — 搬运过程中的踩坑
- `memory/decisions.md` — 架构层面的联合决策
- `memory/bpt-guidance-log.md`（待创建） — 每轮搬运的主题 / 日期 / 成果概要（一行一条）
- 本文件 — 协议本身的迭代

## 六、边界（绝不做，v0.3 修订）

1. **银芯绝不直接访问 BPT 仓库**——即使技术上可行，也必须经守密人转述。
   （黑池 fetch 银芯是反方向的合法拉取，与本条不冲突：银芯被动被读，绝不主动去读。）
2. **银芯绝不向黑池推送**——不 push 代码、不订阅 BPT 仓库、不调用 BPT 的 API；
   通道①的主动方永远是黑池。（v0.2「绝不自动化指导流程」修订为本条：自动化禁令的
   实质是「银芯不得主动越界」，黑池自动 fetch 不在禁令射程内。）
3. **BPT 的内部决策不入银芯档案**——只有守密人亲述认可并搬回银芯的经验才沉淀
   （= 通道② b 回流过滤，本条即其明文依据）。
4. **对话产出不含 BPT 的敏感代码片段**——银芯是公开层（BIAV-SC），不污染。

## 八、BPT 知识架构指导：脊柱↔长尾边界规范（方法论输出·2026-07-05 拷问收敛，守密人裁定 v0.4 纳入）

> 承 §三「方法论输出」。BPT 数据金矿已连全归档（聊天/游戏仓/会议转录/社区/日报/文档库），
> 前沿转向**信息的权重 / 去噪 / 结构化**与**调用**。银芯以自身 OKF / `kb_navigator` 经验，
> 指导 BPT 把金矿组织成「**LLM 总指挥编排 grep + 向量 + OKF 三条腿**」的两层结构。经 2026-07-05
> 六节拷问收敛成下列边界规范，守密人裁定正式纳入协议、经通道① 交付 BPT 参考（**不含任何黑池数据**）。

### 8.1 大原则
知识分两层、LLM 编排：**脊柱（OKF）= 白盒**（可审计 / 带类型 / 可枚举 / 可溯源）；
**长尾（向量）= 黑盒**（模糊语义召回）。两层互补不互斥。

> **零 ML 红线反转（守密人 2026-07-05 裁定 A，见 decisions.md）**：原「向量这一级银芯自身因
> 零 ML 红线不用、该级归 BPT 自建」已**反转**——银芯自建向量腿（Voyage 嵌入，`scripts/kb_vector.py`）
> 作本节「厚锚撑向量」的**活体参照实现**，令 BPT 有得照抄（补上 8.6 唯一抄不到的天窗）。
> **反转是 scoped 的**：白盒脊柱（OKF/kb_index）仍确定性零 ML，只新增**隔离**的 ML 向量长尾腿；
> 二层互补的架构不变。BPT 仍自建自己的向量层（吃自己的私有语料）；银芯向量腿只吃自己**已公开**的
> 社区档案，**§1.1-HC 黑池防火墙无涉**（非黑池数据、方向红线不破）。

### 8.2 建造侧（什么进脊柱、怎么进）

| # | 节 | 规范 |
|---|----|------|
| 1 | **判据** | 关系为唯一标尺：**问关系 / 边界 / 溯源 → 脊柱，问内容 / 找相似 → 长尾**。决策是「有关系」的特例；引用数是提炼触发器、非判据。 |
| 2 | **粒度** | **唤醒体尺寸的持久大实体做节点**（非事件原子）；决策 / 事件 = 挂在实体上的**带日期属性**。一概念一文件、放指针不放本体。 |
| 3 | **实体本体论** | ①功能 / 系统 ②产品 / 项目 / 发行 ③机制 / 玩法 ④长期议题 / 立场 ⑤人 / 外部伙伴（黑池侧·防火墙内）⑥文档（权威被引用者；**既是容器又是实体**——节点持身份 + 边 + 指针、正文留长尾）。 |
| 4 | **挂靠去噪线** | **状态变更 / 持久关系 → 挂脊柱**；评述 / 观察 / 情绪 / 快照 → 沉长尾。测试：进该实体「官方变更日志」还是只是「关于它的闲聊」。 |
| 5 | **提炼机制** | **LLM 自动写入**，靠三承重墙守白盒：**人可见**（出身牌 + 评审流）· **可撤回**（细粒度原子写 + 归属，禁 blob 重写）· **确认态**（`机器草拟·未确认` vs `人已确认`）；**确认惰性、由消费驱动**（热节点先审、死节点不必审）。铁律：LLM 可建白盒，但每笔写入必须可见其出身与确认态——看得见才仍是白盒。 |
| 6 | **厚锚词表（AI 自动识别、人零维护）** | 每个实体节点带**审定别名 / 黑话表**（达芙妮 ≈ 冰系奶妈 / 技能名…），当合流的「厚锚」（节 7 用）。**AI 全自动识别、人零维护**：别名由 AI 从语料自动抽、由 §8.4「锚不到」的**消费失败自动喂大**（非人肉批量建表）。仍守节 5 三墙——每条别名带**出身牌**（从哪条消息 / 哪次推断得出）· **可单条撤回** · `未确认 / 已确认` 态。人只留**近零成本的最终否决**（撤错的一条），不管全表。诚实代价：全自动难免混入认错别名——靠出身牌可抓、未确认态压检索权重、遥测（评判 #2）自动挑高频条供人偶尔一瞄，「该瞄的」由系统挑、非人巡全表。**要点**：「AI 自动识别」与「不失控回黑盒」靠三墙焊住，不靠人力。 |

### 8.3 检索侧（查询来了怎么合流）

| # | 节 | 规范 |
|---|----|------|
| 7 | **合流（厚锚撑向量，非数据级过滤）** | **先锚后扩**——但「扩」**不靠给数据动手脚**：拷问否掉「逐消息贴 entity_id」（几百万条，落地不能）与「顺实体指针圈范围」（聊天/转录按频道归档、实体卡指针够不到只顺嘴提及它的散句，漏掉最值钱的长尾）。改为**把厚锚（节 6 别名表）交给 LLM**：① **扩词召回**——LLM 拿实体别名去查向量，「冰系奶妈大招被削」这类换说法、不含本名的散句也被捞进候选（因脊柱审定过「冰系奶妈 = 达芙妮」）；② **据锚去杂**——LLM 端着精确身份 / 边界读候选，把「其实在讲别的实体」的判掉。脊柱**不焊过滤器**，脊柱**喂 LLM 精确身份 + 词表 + 边界**当护栏（提精度）。答案 = **脊柱托底（可审计）+ 长尾上色（够丰满）**。诚实残差：既无本名、别名又没覆盖的纯散句，交 LLM 当场据锚判 / 判不准则宽搜——模糊长尾不可消的尾巴，不假装白捡。 |

### 8.4 闭环
**锚不到即提炼信号**——检索反哺建造：查询在脊柱锚不到、长尾却一大摞 = 脊柱下一步该长的地方
（接遥测零命中回流）。**权重从策展自然长出**：被查 / 被引用决定先提炼什么。

### 8.5 防火墙红线（硬约束，承 §六）
全为方法论 + 银芯公开工程模式。红线守的是**数据方向**，不是「银芯禁有向量」：**黑池数据、
人 / 会议转录等 BPT 私有语料一律建在 BPT 侧，永不进银芯**（银芯 → 黑池单向，§1.1-HC 不变）。
2026-07-05 反转后银芯自建的向量腿**只吃自己已公开的社区档案**（同源公开数据），与本红线正交、不破防火墙。

### 8.6 银芯可交付的参照实现（公开工程产物，经通道①）
OKF bundle 造法 / `scripts/kb_navigator.py` 导航后端 / **`scripts/kb_vector.py` 向量长尾腿

### 8.7 指令层：路由器不是目录（agent 指令的脊柱↔长尾兑现·2026-07-05 续拷问收敛，守密人裁定 v0.5 纳入）

承 8.1–8.6：脊柱↔长尾不只管**数据金矿怎么建**，也管**agent 指令（BPT 的 CLAUDE.md 等价物）怎么写**。
洞察：**agent 常驻指令里手抄的「目录 / 文件清单 / 每文件长描述」应交给 OKF 脊柱，指令本身缩成「路由器 + 入口」。**

- **判据（三条腿各司其职，勿混为可互替）**：**OKF（脊柱）取代「目录」**（逐文件枚举 + 结构 + 层判定）；
  **grep 不被取代**（关键词命中维度 A/B 实测 KB=grep，纯查串 ripgrep 更便宜）；**向量补「模糊语义」**（2026-07-05 反转后银芯已建参照腿 `kb_vector`，BPT 换自己私有语料自建）。
  三者是三条腿、不是三个可互换的抽屉——只有「目录」这条活从指令移交 OKF。
- **为什么指令不能全删、必留 bootstrap**：**加载时机不对等**——常驻指令每会话自动加载、OKF 按需 fetch。
  你无法从「什么都不知道」bootstrap 出「去调 kb_*」这个动作。故指令必留一张最小卡：**有个知识库 / 入口在哪 / 何时敲哪条腿**（消费路由）。
- **留 / 瘦判据（防误删加载期规则）**：**纯枚举目录 → 瘦成 OKF 指针**；**加载期操作规则 + 权威指定 → 必留**
  （如「某数据层读取铁律」「某档案为唯一权威」——OKF 给你指针、给不了「别把输出层当全量用」这类使用纪律）。测试：删掉它，下一次会话会不会因此读错数据 / 认错权威？会 = 必留。
- **收益**：治指令漂移（手抄目录必烂、还得挂对账测试去追）；指令从「当那本目录」缩成「当路由器」，加载更轻、真相单一（目录只有 OKF 一份）。
- **银芯活体范例（照抄参照）**：CLAUDE.md §5 前言改路由卡、§5.1 长描述表→锚点指针、§6.1 层枚举→`kb_overview` 指针，
  §5.2/§5.3 加载期规则原文留；对账三卫（路径存在 / 反向漂移覆盖 / 日期一致）守其不烂。见 PR #438、`scripts/kb_navigator.py:overview()`。
