## 品牌配色总纲(v3.0 确立)

品牌只有两套配色方案,所有交付物必须择一:

1. **黑金**:深黑底 `#0a0b10` + 金色系 + 暖米文字。默认方案,适合报告、架构文档、图解。
2. **乳白金**:乳白底 + 金色系 + 深墨文字。适合对外物料、打印友好场景。(色值基线见「乳白金方案」节)

**冷蓝黑、青蓝色系一律不进入品牌交付物**。v2.0 的"禁止冷色调"条款维持有效并扩大到功能色以外的一切用色。

**金色提纯**(v3.0 核心变更):旧主金 `#c5a356`/暗金 `#8a7a48` 偏棕发闷,大面积使用时呈现"胶感"。改善方案两条同时执行:
- **换色值**:金色系整体提亮提纯——主金 `#d2ab58`,亮金 `#ecd48d`,暗金 `#9c8850`(仅装饰,不用于正文);
- **控用量**:金色只用于线条、边框、小标题、强调词、装饰元素,**不做大面积填充**。大面积色块一律用底色系(卡片底),让金色以"金属细线"而非"金色塑料面"出现。质感来自克制。

## 核心美学关键词

**苹果官网布局**:大量留白、大标题居中、一屏一主题、呼吸感强、排版驱动。
**手机阅读优先**:正文 ≥ 15pt,辅助文字 ≥ 11pt,表格 ≥ 13pt。**此标准适用于一切交付物形态,包括 PDF 图解的注解文字**;唯一例外是界面线框图内部文字(≥ 9.5pt,视作插图)。
**留白是硬约束**:v2.x 时期交付物被守密人指出"几乎没有留白空间"。自 v3.0 起留白给出具体数值(见「布局原则」),生成后必须自检:单页文字覆盖超过约六成版面即判定过密,拆页或删内容。

### 双色语义系统(v3.0)

架构类文档中需要区分「产品层 vs 技术底层」时:**金 = 产品层/业务层,银白 = SDK/引擎/技术层**。银白呼应「银芯」,全部取自暖米色族,不引入任何蓝色。读者凭颜色直接读出分层归属。

### 功能色(仅用于数据语义,不属于品牌色)

| 用途 | 色值 | 说明 |
|------|------|------|
| 正面/安全 | `#7aad5a` | 好评、安全信号、正面指标 |
| 负面/风险 | `#c25a4a` | 差评、风险预警、负面指标 |

v2.0 的洞察蓝 `#5a8aad` 及 insight-box **废除**(冷色出品牌),原洞察语义由银白系或亮金承担。功能色只出现在数据图表和风险/安全框,不承担品牌视觉。

## 乳白金方案(v3.0,守密人裁定定稿 = 候选浅A「纸白中性」)

**核心原则(第二轮反馈得出):金要凸显,靠的不是金更黄,而是其余一切去黄提灰。** 初版底/卡/墨全在黄调里,与金只剩明度差、没有色相差,守密人反馈"整体偏黄、金不凸显"。修正:底色退到近白微暖、卡片转中性灰白、墨转中性,金成为画面唯一高饱和暖色。饱和度分布:底 0.02 / 卡 0.03 / 墨 0.09 / 主金 0.74 / 深金 0.85。2026-07-12 三候选对照(A 纸白中性 / B 瓷白白卡 / C 高白高对比)后守密人裁定 A 为正式方案,B/C 落选归档于 palette-compare 对照图。

| 用途 | 色值 |
|------|------|
| 背景 | `#faf9f6`(近白微暖;初版 `#f7f3ea` 偏黄已弃) |
| 卡片底(一级) | `#f1efe9`(中性灰白) |
| 卡片底(二级) | `#e9e6de` |
| 主金(线条/标签) | `#9a7a28`(高饱和,3.8:1) |
| 深金(标题/强调文字) | `#8c6a15`(高饱和,4.8:1) |
| 金装饰 | `#a8862e`(仅封面副标题等装饰) |
| 正文 | `#2b2a27`(中性墨,13.6:1) |
| 辅助文字 | `#605d55`(中性灰,6.2:1) |
| 低层级辅助 | `#8d8a82`(页脚/元信息,3.3:1) |
| 墨系语义(替代银白) | `#4a4841` 深 / `#706d64` 浅(中性,技术层语义) |
| 金色边框 | `rgba(154,122,40,.55)` |
| 中性边框 | `rgba(43,42,39,.13)` |
| 金色框底 | `rgba(154,122,40,.07)` |
| 墨系框底 | `rgba(74,72,65,.05)` |
| 负面(浅底) | 边框 `rgba(164,70,58,.5)`,文字 `#8d4034` |
| 装饰菱形 | `#d6d2c6` |

浅底校验规则:正文 ≥ 7:1,标题/强调 ≥ 4.5:1,标签与装饰 ≥ 3:1(WCAG 换算);另加**饱和度检查**——金以外的底/卡/中性色饱和度须 ≤ 0.10,保证金是唯一高饱和色。字体、布局、组件规则与黑金方案完全一致;暗↔亮转换时金色角色互换(深底亮金做标题,浅底深金做标题)。

## 字体

| 用途 | 字体 | 字重 | 最小字号 |
|------|------|------|------|
| 章节标题 | Noto Serif CJK SC / Noto Serif SC | SemiBold (600) 或 Bold (700) | 22pt |
| h3 小标题 | Noto Sans CJK SC | Bold (700) | 16pt |
| 正文 | Noto Sans CJK SC | Medium (500) | 15pt |
| 图解注解/说明 | Noto Sans CJK SC | Medium (500) | 15pt(v3.0 明确:注解即正文,不得降级) |
| 表格内容 | Noto Sans CJK SC | Medium (500) | 13pt |
| 表头 | Noto Sans CJK SC | Bold (700) | 13pt |
| chip 标签/组件小字 | Noto Sans CJK SC | Medium (500) | 11pt |
| 页码/注释 | Noto Sans CJK SC | Regular (400) | 10pt |
| 线框图内部文字 | Noto Sans CJK SC | Medium/Bold | 9.5pt(全规范唯一例外,视作插图) |
| 封面主标题 | Noto Serif CJK SC | Bold (700) | 32pt |
| 封面英文副标题 | Noto Sans CJK SC | Light (300) | 11pt |
| 数值指标(卡片) | Noto Serif CJK SC | Bold (700) | 20pt |

**关键原则**:
- 必须用粗字重。Regular 仅用于页码和注释,标题和强调处永远用 SemiBold 或 Bold。
- 正文用 Medium (500) 而非 Regular (400),深色背景上可读性显著更好。
- 表格正文也用 Medium (500)。

## 布局原则

- **一屏一主题**:每个章节 page-break-before: always;HTML 长滚动页同理,每屏只承载一个主题模块
- **留白硬性数值**(v3.0):
  - PDF 页边距:上 26mm、左右 22mm、下 22mm(由 v2.0 的 18/16mm 放宽)
  - 章节标题区与正文间距 ≥ 12mm;同页模块之间 ≥ 9mm;图文并排列间距 ≥ 9mm
  - 封面除文字外全部留白,不放图形组件
  - 自检:单页文字覆盖率约六成为上限,超出即拆页
- **行高 2.0**:正文 line-height: 2,表格与组件内 1.7~1.9
- **段落间距**:4mm margin-bottom
- **列表缩进**:5~6mm padding-left,列表项前用 ◆(主金,7pt)
- **章节标题结构**:eyebrow 小字(11pt,letter-spacing 5pt,主金)→ 标题 → section-rule(40px × 2px,主金)
- **避免孤页**:orphans: 3, widows: 3,关键区块 page-break-inside: avoid
- **对齐**:正文 justify 或左对齐,封面与引言居中
- **页脚**:左侧文档名、右侧页码「NN / NN」,10pt,`#7a7468`,letter-spacing 1.5pt
- **余白吸收**:页面内容与页脚之间用装饰菱形 ◇ ◇ ◇ 收尾

## 表格规范

```css
table { width: 100%; border-collapse: collapse; font-size: 13pt; }
th { background: #1a1915; color: #ecd48d; font-weight: 700; padding: 3mm 3mm; border: 0.5px solid rgba(242,237,226,.14); }
td { padding: 2.5mm 3mm; border: 0.5px solid rgba(242,237,226,.14); color: #b8ad9c; vertical-align: top; line-height: 1.7; font-weight: 500; }
tr:nth-child(even) td { background: rgba(210,171,88,0.04); }
tr:nth-child(odd) td { background: rgba(10,11,16,0.5); }
td b { color: #ecd48d; }
```

- 表格内长文本必须用 Paragraph 对象(reportlab)或正常 `<td>` 文本流(HTML)
- 允许表格跨页
- 表格注释 11pt,`#7a7468`,居中,letter-spacing 1pt

## 特殊组件

### 高亮框 (highlight-box)
```css
border-left: 4px solid #d2ab58;
padding: 5mm 6mm;
background: rgba(210,171,88,0.06);
color: #ecd48d;
font-weight: 500; font-size: 15pt; line-height: 1.9;
```

### 风险框 (risk-box) / 安全框 (safe-box)
同 v2.0,边框与文字色分别用 `#c25a4a` / `#7aad5a` 系,底 6% 透明度。字号 15pt。

### 银白框 (silver-box)(v3.0 新增,替代 insight-box)
```css
border: 0.5pt solid rgba(242,237,226,.4);
border-radius: 2.5mm;
padding: 5mm 6mm;
background: rgba(242,237,226,.05);
/* 标题 #e8e4da,正文 #b8ad9c */
```
用于 SDK/引擎/技术层语义、中性洞察。

### 双栏对照框 (split-box)(v3.0 新增)
「已有 vs 要做」类对照。左银白框(技术/既有)、右金框(产品/待做)并排,各带 letter-spacing 小标题(11pt)+ ◆ 列表(13pt)。列间距 ≥ 5mm。

### 分层责任图 (layer-diagram)(v3.0 新增)
每层 = 左侧窄列(层名 28mm,Serif Bold 13pt,按层语义着色)+ 右侧 chip 流。层间用居中「▲ 动词」行连接(主金,11pt,letter-spacing 3pt)。chip:11pt,padding 1.2mm 4mm,圆角 5mm,边框与文字按金系/银白系/中性。

### 界面线框图 (ui-mock)(v3.0 新增)
样式化线框展示界面设计意图。顶栏(三圆点 + 9.5pt 标题)+ 内容区(对话气泡 10.5pt / 卡片 10pt / 抽屉 10pt)。底色 `#131210`,内部元素 `#1a1915`,强调边框金系。宽度 ≤ 版心 45%,与注解并排;注解字号不受线框图影响,仍 ≥ 15pt。

### 锚点对比卡 (anchor-compare)(v3.0 新增)
两卡并排做概念对比。每卡:小标签(11pt,letter-spacing)→ 锚点词(Serif Bold 19pt 居中)→ 上下细线夹住的 ◆ 特征列表(13.5pt)→ 一句结论。被推荐方金系,对照方银白系。
**禁用圈中圈式图形**(如中心圆+卫星圆的轨道图):小圆内塞文字必然低于字号标准,且多圆相邻产生视觉噪声,实测严重影响可读性。概念对比一律用文字列表承载。

### 流程步骤条 (flow-steps)(v3.0 新增)
横向等宽步骤框,「→」连接(主金)。每步:Serif 步骤名 14pt 亮金 + 11pt 说明。步骤数 ≤ 5。

### 总览卡片网格 (overview-grid)
```css
.overview-card {
  background: #131210; border: 0.5px solid rgba(242,237,226,.14);
  border-radius: 3px; padding: 4mm 5mm; text-align: center;
}
.overview-card .num { font-family: Noto Serif SC; font-weight: 700; font-size: 20pt; }
.overview-card .label { font-size: 11pt; color: #7a7468; }
```
数值色:正面 `#7aad5a`,负面 `#c25a4a`,中性 `#ecd48d`,技术 `#e8e4da`。

### 主题卡片 (theme-card)
底 `#131210`,边框中性,主题名 15pt 亮金,计数 11pt,引文 12pt `#b8ad9c` 带金色左线。

### 制作人引言 (producer-quote)
Serif,15~21pt,亮金,居中,line-height 2~2.2。可用作章节命题引言(「」括起,21pt),前后各留 ≥ 10mm。

### 装饰菱形 (deco-diamond)
`text-align:center; color:#3a382c; font-size:10pt; letter-spacing:6pt;` 内容 ◇ ◇ ◇。

## 封面规范

```css
.cover { text-align: center; /* 垂直居中整页 */ }
.cover-label { font-weight: 300; font-size: 11pt; color: #7a7468; letter-spacing: 7pt; }
.cover h1 { font-family: Noto Serif SC; font-weight: 700; font-size: 31~32pt; color: #ecd48d; letter-spacing: 3pt; line-height: 1.7; }
.cover-sub { font-size: 15pt; color: #9c8850; letter-spacing: 1pt; line-height: 2.1; }
.cover-line { width: 60px; height: 2px; background: #d2ab58; margin: 14mm auto; }
.cover-meta { font-size: 12pt; color: #7a7468; line-height: 2.3; }
```

结构自上而下:标签(DOC TYPE)→ 主标题 → 装饰线 → 副标题(可用一句话命题)→ 元信息。整页只有文字,垂直居中。

## PDF 生成

**首选 weasyprint**。

```python
from weasyprint import HTML
HTML('report.html').write_pdf('output.pdf')
```

v3.0 实操要点:
- **禁止「固定高度页面盒 + 绝对定位页脚」做法**:内容一涨即压脚出框。正确做法:@page 设真实边距(`margin: 24mm 20mm 26mm; background: #0a0b10`,背景自动铺满全页含边距),页脚用 @page 边距盒(`@bottom-left` / `@bottom-right`,content + counter(page)),章节用 page-break-before 自然分页——内容物理上不可能越界,涨了就流到下一页
- 封面用命名页(`page: cover`)抑制页脚
- 含长 ASCII 串或 letter-spacing 的文本块加 3~4mm 右侧 padding 防溢缓冲
- 布局用 flex,不用 grid(weasyprint 的 grid 支持不稳)
- 环境有本地 Noto CJK 字体时直接用 `Noto Sans CJK SC` / `Noto Serif CJK SC` 族名,无需 CDN
- 生成后必须 pdf2image 渲染抽查(封面、一个内容页、最密一页);目视通道不可用时,用像素统计兜底:①底部 2% 行亮像素占比检测溢出,②B 通道减 R 通道检测冷蓝残留,③版心亮像素占比粗估文字覆盖率

**reportlab 仅在需要程序化动态生成时使用**:OTC 不可用需 .otf/.ttf 单体;Variable Font 用 fonttools.varLib.mutator 生成静态字重;深色背景在 onPage 回调手动绘制全页矩形。

## HTML 交付物规范

- Google Fonts CDN 加载 Noto Sans SC / Noto Serif SC(容器环境可用本地 CJK)
- 单文件自包含,深色 `#0a0b10` 设在 body
- vanilla JS,不依赖框架
- 卡片底 `#131210`,嵌套层 `#1a1915`;边框中性 `rgba(242,237,226,.14)`,强调 `rgba(210,171,88,.32)`
- 动效克制:仅入场 fade(0.7s),尊重 prefers-reduced-motion
- 留白同样适用:每屏一主题,版心 ≤ 880px,section 间距 ≥ 72px
- **字号同样适用手机标准**:正文 15px 起,不因网页形态降级

## 长图导出规范(v3.0 新增,第二轮修正)

PDF 转长图(多页竖拼)用于手机传阅/钉钉分享。清晰度的决定因素是**像素预算**,不是压缩质量:
- **300dpi 原生直出,不缩放**(A4 → 2480px 宽)。此时 15pt 字高 ≈ 62px,中文笔画余量充足;先前按 1290px"手机物理宽度"导出的做法废除——那样 15pt 字只有 ≈32px,先天不足,任何压缩参数都救不回
- 渲染与目标尺寸一致,避免"高 dpi 渲染再缩放"引入的重采样模糊
- JPEG quality 90(体积约 1.5MB/三页,可接受);压缩以文字边缘无可见劣化为底线
- 页间不留缝,背景色与文档底色一致

## PPT 规范

- 纯黑背景 `#0a0b10`
- 标题字体 Georgia serif(PPT 环境下的 Noto Serif 替代)
- 配色层级:亮金 `#ecd48d` 标题 → 主金 `#d2ab58` 强调 → 暖米 `#f2ede2` 正文 → 暗米 `#b8ad9c` 辅助
- 一页一主题,大量留白
- 不使用渐变、阴影、动画转场

-----

## 禁止事项

- 不使用 emoji(任何交付物)
- 不使用冷色调:冷蓝黑底、青蓝功能色、任何 B>R 的中性色(v3.0 重申并扩大)
- 金色不做大面积填充(v3.0 新增,防"胶感")
- 不使用细字重标题(Light / Thin)
- 不使用默认白底黑字(乳白金方案除外,该方案底色为品牌乳白非纯白)
- 不使用系统字体(Arial、Helvetica、SimSun)
- 不使用 bullet point 圆点(用 ◆ 替代)
- 注解、说明类文字不得以"辅助文字"名义降到 15pt 以下(v3.0 新增)
